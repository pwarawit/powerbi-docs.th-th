# <a name="drill-mode-in-a-visualization-in-power-bi"></a>โหมดการดูรายละเอียดแนวลึกในการแสดงภาพใน Power BI

## <a name="drill-requires-a-hierarchy"></a>การดูรายละเอียดแนวลึก จำเป็นต้องมีลำดับชั้น
เมื่อวิชวลมีลำดับชั้น คุณสามารถเจาะลงไปเพื่อดูรายละเอียดเพิ่มเติม ตัวอย่างเช่น คุณอาจมีการแสดงภาพที่นับจำนวนเหรียญรางวัลโอลิมปิก โดยมีลำดับชั้นที่สร้างขึ้นจาก ชนิดกีฬา ประเภทกีฬา และรายการแข่งขัน ตามค่าเริ่มต้น การแสดงภาพจะแสดงจำนวนเหรียญรางวัลตามชนิดกีฬา - ยิมนาสติก, สกี, กีฬาทางน้ำ และอื่น ๆ แต่เนื่องจากมีลำดับชั้น เลือกองค์ประกอบหนึ่งของวิชวล (เช่น แท่ง เส้น หรือฟอง) จะแสดงรูปภาพที่มีรายละเอียดเพิ่มขึ้นเรื่อย ๆ เลือกองค์ประกอบ**กีฬาทางน้ำ** เพื่อดูข้อมูลสำหรับประเภทกีฬา ว่ายน้ำ กระโดดน้ำ และโปโลน้ำ  เลือกองค์ประกอบ**กระโดดน้ำ** เพื่อดูรายละเอียดสำหรับการแข่งขัน กระดานสปริงบอร์ด แพลตฟอร์ม และ กระโดดน้ำซิงโครไนซ์

คุณสามารถเพิ่มลำดับขั้นไปยังการรายงานของคุณเองได้ แต่ต้องไม่ใช่การรายงานอื่นที่แชร์กับคุณ
ไม่แน่ใจว่าการแสดงภาพ Power BI ไหนมีลำดับชั้น?  โฮเวอร์เหนือการแสดงภาพ และถ้าคุณเห็นตัวควบคุมเพื่อดูรายละเอียดเหล่านี้ การแสดงภาพของคุณมีลำดับชั้น

![ระดับการดูรายละเอียดแนวลึก](./media/end-user-drill/power-bi-drill-icon4.png)  ![การเปิดและปิดการดูรายละเอียดแนวลึก](./media/end-user-drill/power-bi-drill-icon2.png) ![ไอคอนดูรายละเอียดแนวลึกทุกเขตข้อมูลในครั้งเดียว](./media/end-user-drill/power-bi-drill-icon3.png)
 ![ไอคอนดูข้อมูลสรุป](./media/end-user-drill/power-bi-drill-icon5.png) ![ไอคอนขยายด้านล่าง](./media/end-user-drill/power-bi-drill-icon6.png)  

วันที่คือประเภทเฉพาะของลำดับชั้น เมื่อคุณเพิ่มเขตข้อมูลวันที่ให้กับการแสดงภาพ Power BI จะเพิ่มลำดับชั้นเวลา ที่มี ปี, ไตรมาส, เดือน, วัน โดยอัตโนมัติ สำหรับข้อมูลเพิ่มเติม โปรดไปยัง [ลำดับชั้นที่แสดงและรูปแบบการดูรายละเอียดแนวลึก](../guided-learning/visualizations.yml?tutorial-step=18) หรือดูวิดีโอด้านล่าง

## <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น

1. ในบริการของ Power BI หรือ Desktop การดูรายละเอียดต้องการการแสดงภาพที่มีลำดับชั้น 
   
2. ในการปฏิบัติสำหรับขั้นตอนนี้ โปรด[เปิดตัวอย่างการวิเคราะห์การค้าปลีก](../sample-datasets.md) และสร้างแผนภูมิต้นไม้ที่แสดง **หน่วยรวมปีนี้** (ค่าแสดง) จำแนกตาม **ดินแดน** **เมือง** **รหัสไปรณีย์** และ **ชื่อ** (กลุ่ม)  แผนภูมิต้นไม้นี้มีลำดับขั้นที่มาจากดินแดน เมือง รหัสไปรษณีย์ และชื่อเมือง แต่ละดินแดน มีหนึ่งหรือหลายเมือง แต่ละเมือง มีหนึ่งหรือหลายรหัสไปรษณีย์ และต่อไปเรื่อย ๆ ตามค่าเริ่มต้น การแสดงภาพจะแสดงเฉพาะข้อมูลดินแดน เนื่องจาก*ดินแดน*ปรากฏเป็นรายการแรกในลำดับชั้น
   
   ![เลือก "ดินแดน"](media/end-user-drill/power-bi-hierarcy-list.png)

2. ควรทำความเข้าใจในไอคอนการดูรายละเอียดต่าง ๆ ที่ทำงานร่วมกัน เพราะอาจทำให้เกิดความสับสนได้ ดังนั้นลองเลือกกรองข้อมูลแผนภูมิต้นไม้ให้แสดงเพียง 2 ดินแดนที่เล็กกว่า **KY** และ **TN** เลือกแผนภูมิต้นไม้และดู**ตัวกรองระดับการแสดง** ขยาย**ดินแดน** และเลือก **KY** และ **TN**

    ![ตัวกรองสำหรับ KY และ TN](./media/end-user-drill/power-bi-filter.png)    

   ในตอนนี้ ดินแดนทั้งสองจะปรากฏบนแผนภูมิต้นไม้

   ![ไอคอนดูรายละเอียดคู่](./media/end-user-drill/power-bi-territories.png)

## <a name="three-ways-to-access-the-drill-features"></a>สามวิธีในการเข้าถึงคุณลักษณะการดูรายละเอียด
มีตัวเลือกมากมายสำหรับการเข้าถึงการดูรายละเอียดแนวลึก ข้อมูลสรุป และขยายคุณสมบัติในการแสดงผลภาพที่มีลำดับขั้น บทความนี้จะแสดงวิธีการใช้ตัวเลือกแรกทางด้านล่าง เมื่อคุณเข้าใจพื้นฐานของการดูรายละเอียดแนวลึกและการขยายเรียบร้อยแล้ว ทั้งสามวิธีนี้จะมีการดำเนินการเหมือนกัน ลองทำต่อไป และเลือกวิธีที่คุณชื่นชอบมากที่สุด

- โฮเวอร์เหนือการแสดงภาพเพื่อดู และใช้ไอคอน  

    ![เส้นทางการดูรายละเอียด](./media/end-user-drill/power-bi-hover.png)

- คลิกขวาที่การแสดงภาพเพื่อเปิดเผย และใช้เมนู
    
    ![เมนูบริบท](./media/end-user-drill/power-bi-drill-menu.png)

- จากแถบเมนู Power BI เลือกปุ่ม**สำรวจ**

   ![การเลือกปุ่มสำรวจ จะแสดงไอคอนดูรายละเอียดและตัวเลือก](media/end-user-drill/power-bi-explore.png)

## <a name="drill-pathways"></a>เส้นทางการดูรายละเอียด
### <a name="drill-down"></a>การดูรายละเอียดแนวลึก
คุณมีหลายวิธีที่จะดูรายละเอียดการแสดงภาพของคุณ ***การดูรายละเอียดแนวลึก*** จะนำคุณไปยังระดับถัดไปในลำดับชั้น ดังนั้นถ้าคุณกำลังดูระดับ**ดินแดน** คุณสามารถเข้าถึงรายละเอียดในระดับเมือง จากนั้นระดับรหัสไปรษณีย์ และสุดท้ายคือระดับชื่อได้ แต่ละขั้นตอนในเส้นทางจะแสดงข้อมูลใหม่ให้คุณ

![เส้นทางการดูรายละเอียด](./media/end-user-drill/power-bi-drill-path.png)

### <a name="expand"></a>ขยาย

***ขยาย*** เป็นการเพิ่มระดับของลำดับชั้นในมุมมองปัจจุบัน หากคุณต้องการดูระดับ**ดินแดน** คุณสามารถขยายและเพิ่มเมือง รหัสไปรษณีย์ และรายละเอียดชื่อไปยังแผนภูมิต้นไม้ได้ แต่ละขั้นตอนในเส้นทางแสดงข้อมูลเดียวกัน และเพิ่มข้อมูลใหม่อีกหนึ่งระดับ

![เส้นทางการขยาย](./media/end-user-drill/power-bi-expand-path.png)

คุณยังสามารถเลือกว่าจะดูรายละเอียดแนวลึก หรือขยายทีละหนึ่งเขตข้อมูล หรือทุกเขตข้อมูลพร้อมกันก็ได้ 

## <a name="drill-down-all-fields-at-a-time"></a>ดูรายละเอียดแนวลึกทุกเขตข้อมูลพร้อมกัน

1. มาเริ่มที่ระดับบนสุดของแผนภูมิต้นไม้ที่แสดงข้อมูลสำหรับ KY และ TN ขยายแผนภูมิต้นไม้ของคุณโดยการเลือกจุดจับจุดหนึ่ง แล้วลากไปด้านขวา 

    ![แผนภูมิต้นไม้จะแสดงสองรัฐ](./media/end-user-drill/power-bi-drill-down.png) .

2. วิธีดูรายละเอียดแนวลึก***ทุกเขตข้อมูลในครั้งเดียว*** ให้เลือกลูกศรคู่ที่มุมซ้ายบนของการแสดงผลภาพ![ไอคอนดูรายละเอียดคู่](./media/end-user-drill/power-bi-drill-icon3.png) ในตอนนี้แผนภูมิต้นไม้ของคุณจะแสดงข้อมูลรัฐ Kentucky และ Tennessee 

    ![ไอคอนดูรายละเอียดคู่](./media/end-user-drill/power-bi-drill-down1.png)
   
5. ดูรายละเอียดแนวลึกอีกครั้งสำหรับลำดับชั้นของข้อมูลรหัสไปรษณีย์

    ![ไอคอนดูรายละเอียดคู่](./media/end-user-drill/power-bi-drill-down2.png)

3. หากต้องการดูรายละเอียดสำรอง ให้เลือกลูกศรชี้ขึ้นที่มุมบนซ้ายของการแสดงภาพ ![ไอคอนดูข้อมูลเลื่อนขึ้นอีกหนึ่งระดับ](./media/end-user-drill/power-bi-drill-icon5.png).


## <a name="drill-down-one-field-at-a-time"></a>ดูรายละเอียดแนว﻿ลึกทีละเขตข้อมูล
วิธีนี้จะใช้ไอคอนดูรายละเอียด﻿แนวลึกบริเวณมุมบนขวาของการแสดงภาพ 

1. เลือกไอคอนดูรายละเอียดเพื่อ ![เปิดใช้งานการดูรายละเอียดแนวลึก](./media/end-user-drill/power-bi-drill-icon2.png) ในตอนนี้คุณมีตัวเลือกที่จะดูรายละเอียด***ทีละเขตข้อมูล*** 
   
   ![ชี้ลูกศรไปที่ไอคอนเปิด/ปิดการดูรายละเอียดแนวลึก](media/end-user-drill/power-bi-drill-icon-new.png)

   ถ้าคุณปิดใช้งานการดูลายละเอียดแนวลึก การเลือกองค์ประกอบที่แสดง (เช่น แถบ ฟองสบู่ หรือใบไม้) จะไม่แสดงรายละเอียดแนวลึก แต่จะกรองข้ามแผนภูมิอื่น ๆ บนหน้ารายงานแทน

2. เลือก*ใบไม้*สำหรับ **TN** ในตอนนี้แผนภูมิต้นไม้ของคุณจะแสดงเมืองทั้งหมดใน Tennessee ที่มีร้านค้าอยู่ 

    ![แผนภูมิต้นไม้จะแสดงข้อมูลสำหรับ Tennesee เท่านั้น](media/end-user-drill/power-bi-drill-down-one1.png)

2. ในขั้นตอนนี้ คุณสามารถดูรายละเอียดแนวลึกสำหรับ Tennesee ได้ หรือดูเมืองใดเมืองหนึ่งในนั้นโดยเฉพาะ หรือดูแบบขยายแทน (ลองดู **วิธีขยายเขตข้อมูลทั้งหมดในครั้งเดียว** ที่ด้านล่าง) ลองดูรายละเอียดแนวลึกทีละเขตข้อมูล  เลือก **Knoxville, TN** (เมือง Knoxville รัฐ Tennessee) แผนภูมิต้นไม้ของคุณจะแสดงรหัสไปรษณีย์สำหรับร้านค้าที่ Knoxville ของคุณ 

   ![แผนภูมิต้นไม้จะแสดง 37919](media/end-user-drill/power-bi-drill-down-one2.png)

    สังเกตอีกครั้งว่าชื่อเรื่องจะเปลี่ยนทุกเมื่อที่คุณดูรายละเอียดแนวลึกและสำรองข้อมูล  

## <a name="expand-all-and-expand-one-field-at-a-time"></a>ขยายทั้งหมด และขยายหนึ่งทีละเขตข้อมูล
การมีแผนภูมิต้นไม้ที่แสดงเพียงรหัสไปรษณีย์ไม่ได้ให้ข้อมูลมากเท่าไหร่  ดังนั้นเรามาขยายลงหนึ่งระดับในลำดับชั้น  

1. ด้วยแผนภูมิต้นไม้ที่ถูกเลือกไว้ก่อนแล้ว เลือกไอคอน*ขยายลง* ![ไอคอนขยายลง](./media/end-user-drill/power-bi-drill-icon6.png) แผนภูมิต้นไม้ของคุณตอนนี้จะแสดง 2 ลำดับชั้น: คือ รหัสไปรษณีย์ และชื่อร้านค้า 

    ![แสดงรหัสไปรษณีย์และชื่อร้านค้า](./media/end-user-drill/power-bi-expand1.png)

2. หากต้องการดูข้อมูลเกี่ยวกับ Tennesee ทั้ง 4 ลำดับชั้น ให้เลือกลูกศรขึ้นจนกว่าคุณไปถึงระดับที่สอง **จำนวนหน่วยรวมปีนี้ที่จำแนกตามดินแดนและเมือง** ของแผนภูมิต้นไม้ของคุณ 

    ![แผนภูมิต้นไม้แสดงข้อมูลทั้งหมดเกี่ยวกับ Tennesee](media/end-user-drill/power-bi-drill-down-one1.png)


3. โปรดตรวจสอบให้แน่ใจว่าการดูรายละเอียดแนวลึกยังเปิดใช้งานอยู่ ![รายละเอียดแนวลึกเปิดใช้งาน](./media/end-user-drill/power-bi-drill-icon2.png) และเลือกไอคอน*ขยายลง* ![ไอคอนขยายลง](./media/end-user-drill/power-bi-drill-icon6.png) แผนภูมิต้นไม้ของคุณในตอนนี้จะแสดงรายละเอียดเพิ่มเติม ซึ่งหมายความว่า นอกจากการแสดงผลแค่เมืองและรัฐแล้ว ยังแสดงรหัสไปรษณีย์อีกด้วย 

    ![ไอคอนดูรายละเอียดคู่](./media/end-user-drill/power-bi-expand-one3.png)

## <a name="understanding-the-hierarchy-axis-and-hierarchy-group"></a>ทำความเข้าใจแกนลำดับชั้นและกลุ่มลำดับชั้น
คุณสามารถนึกถึง แกนลำดับชั้นและกลุ่มลำดับชั้น ว่าเป็นกลไกที่คุณสามารถเพิ่มและลด ความละเอียดของข้อมูลที่คุณต้องการดู ข้อมูลใด ๆ ที่สามารถจัดเป็นประเภทและประเภทย่อย สามารถมีลำดับชั้นได้ ซึ่งแน่นอน รวมไปถึงวันที่และเวลา

คุณสามารถสร้างการแสดงภาพใน Power BI ที่มีลำดับชั้น โดยการเลือกหนึ่งหรือหลายเขตข้อมูล เพื่อเพิ่มไปยัง**แกน**หรือ**กลุ่ม** พร้อมกับข้อมูลคุณที่ต้องการตรวจสอบ เป็นเขตข้อมูลใน**ค่า** คุณจะทราบว่าข้อมูลของคุณมีลำดับชั้น ถ้าไอคอน*โหมดดูรายละเอียด* ปรากฎบนมุมบนซ้ายและขวาของการแสดงภาพของคุณ 

โดยหลัก ๆ แล้ว สามารถคิดว่าลำดับชั้นของข้อมูลมีสองชนิด:
- ข้อมูลวันที่และเวลา - ถ้าคุณมีเขตข้อมูลชนิด วันที่เวลา คุณมีข้อมูลที่มีลำดับชั้นอยู่แล้ว Power BI สร้างลำดับชั้นสำหรับเขตข้อมูลใดก็ได้ ถ้าข้อมูลมีค่าที่สามารถแยกให้มีโครงสร้าง [วันที่เวลา](https://msdn.microsoft.com/library/system.datetime.aspx) คุณเพียงแค่เพิ่มเขตข้อมูล วันที่เวลา ไปยัง**แกน**หรือ**กลุ่ม**
- ข้อมูลที่จำแนกประเภท - หากข้อมูลของคุณมาจากคอลเลกชันที่ประกอบด้วยคอลเลกชันย่อย หรือมีแถวข้อมูลที่แชร์ค่าร่วมกัน ถือว่าคุณมีข้อมูลแบบลำดับชั้น

Power BI ให้คุณสามารถขยายทีละหนึ่งชุดย่อย หรือชุดย่อยทั้งหมด คุณสามารถดูรายละเอียดข้อมูลของคุณ เพื่อดูชุดย่อยเดียวในแต่ละระดับ หรือดูทุกชุดย่อยพร้อมกันในแต่ละระดับ ตัวอย่างเช่น คุณสามารถดูรายละเอียดสำหรับปีที่ระบุ หรือดูผลลัพธ์ทั้งหมดสำหรับแต่ละปีเมื่อคุณลงมาตามลำดับชั้น ในทางกลับกัน คุณสามารถดูข้อมูลสรุปในลักษณะเดียวกัน

ส่วนต่อไปนี้อธิบายการดูรายละเอียดข้อมูล จากมุมมองสูงสุด มุมมองตรงกลาง และมุมมองต่ำสุด

### <a name="hierarchical-data-and-time-data"></a>ข้อมูลลำดับชั้น วันที่และเวลา
สำหรับตัวอย่างนี้ ทดลองทำตามด้วย[ตัวอย่างการวิเคราะห์การค้าปลีก](../sample-datasets.md) และสร้างการแสดงภาพแผนภูมิคอลัมน์แบบเรียงซ้อน ที่ดู**เดือน** (แกน) ตาม**ยอดขายรวม** (ค่า)  

แม้ว่าเขตข้อมูลของแกนจะเป็น**เดือน** ประเภท**ปี**ก็ยังถูกสร้างใน**แกน**ด้วย เนื่องจาก Power BI สร้างโครงสร้างวันที่และเวลาแบบเต็มสำหรับทุกค่าที่อ่าน ด้านบนสุดของลำดับชั้นแสดงข้อมูลสำหรับปี

![แถบบาร์เดี่ยวจะแสดงข้อมูลที่จัดกลุ่มตามปี](media/end-user-drill/power-bi-hierarchical-axis-datetime-1.png)

ในโหมดดูรายละเอียดแนวลึก ให้คลิกที่แท่งในแผนภูมิเพื่อดูข้อมูลลึกลงไปอีกหนึ่งลำดับชั้น คุณจะเห็นแท่งสามแท่ง สำหรับข้อมูลของไตรมาส จากนั้น ที่กลุ่มไอคอนมุมบนซ้าย เลือก**ขยายทั้งหมดลงหนึ่งระดับในลำดับชั้น** แล้วทำเหมือนเดิมอีกครั้ง เพื่อลงไปถึงระดับต่ำสุดของลำดับชั้น ซึ่งแสดงผลลัพธ์สำหรับแต่ละเดือน

![แผนภูมิแท่งเพื่อดูแถบในแต่ละเดือน](media/end-user-drill/power-bi-hierarchical-axis-datetime-2.png)

นอกเหนือจากการแสดงภาพ เราสามารถเห็นลำดับชั้นในข้อมูลที่แสดงสำหรับแต่ละรายงานได้ ตารางต่อไปนี้แสดงผลลัพธ์ของ**ดูข้อมูล** ในรายงานที่ดูรายละเอียดเดือนเดียวหรือทุกเดือน 

สังเกตว่า ข้อมูลเหมือนกันทั้งรายงานไตรมาสและรายงานปี แต่เพื่อคุณดูรายละเอียดถึงระดับที่กำหนดใน**ค่า** คุณจะเห็นว่ารายงานมีความเฉพาะเจาะจงมากขึ้น และรายงาน "ทุกเดือน" มีข้อมูลมากกว่า

### <a name="hierarchical-category-data"></a>ข้อมูลลำดับชั้น ประเภท
ข้อมูลที่มีรูปแบบจากคอลเลกชันและคอลเลกชันย่อย จะจำแนกเป็นลำดับชั้น ตัวอย่างที่ดีตัวอย่างหนึ่งคือ ข้อมูลตำแหน่งที่ตั้ง พิจารณาตารางในแหล่งข้อมูลที่มีคอลัมน์ ประเทศ รัฐ เมือง และรหัสไปรษณีย์ ข้อมูลที่มี ประเทศ รัฐ และเมือง เหมือนกัน ถือว่ามีลำดับขั้น

สำหรับตัวอย่างนี้ ลองทำตามด้วย[ตัวอย่างการวิเคราะห์การค้าปลีก](../sample-datasets.md) สร้างภาพแผนภูมิคอลัมน์แบบเรียงซ้อนที่แสดง**หน่วยรวมปีนี้** (ค่าแสดง) จำแนกตาม**ดินแดน** **เมือง** **รหัสไปรษณีย์** และ**ชื่อ** (กลุ่ม)  

![แผนภูมิแท่งแสดงจำนวนหน่วยรวมปีนี้ตามดินแดน](media/end-user-drill/power-bi-hierarchical-axis-category-1.png)

คุณควรอยู่ในระดับต่ำสุดของลำดับชั้น ซึ่งแสดงผลลัพธ์สำหรับ ดินแดน เมือง และ รหัสไปรษณีย์

## <a name="considerations-and-limitations"></a>ข้อควรพิจารณาและข้อจำกัด
* ถ้าการเพิ่มเขตข้อมูลวันที่ ไม่สร้างลำดับชั้น เป็นไปได้ว่าเขตข้อมูล "วันที่" ไม่ได้ถูกบันทึกเป็นชนิดวันที่จริง ๆ ถ้าคุณเป็นเจ้าของชุดข้อมูล เปิดในมุมมอง*ข้อมูล*ใน Power BI Desktop เลือกคอลัมน์ที่มีวันที่ และในแท็บการวางรูปแบบ เปลี่ยน**ชนิดข้อมูล**เป็น**วันที่** หรือ**วันที่/เวลา** ถ้ารายงานมีแชร์ให้กับคุณ ติดต่อเจ้าของเพื่อขอให้เปลี่ยนแปลง  
  
  ![เลือกมุมมองข้อมูล และดูที่ชนิดของวันบริเวณด้านบนขวา](media/end-user-drill/power-bi-change-data-type2.png)
