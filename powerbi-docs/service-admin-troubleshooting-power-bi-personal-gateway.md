# <a name="troubleshooting-power-bi-gateway---personal"></a>แก้ไขปัญหาเกตเวย์ Power BI - ส่วนบุคคล
ส่วนต่อไปนี้ไปปัญหาทั่วไปบางอย่างคุณอาจมาถึงเมื่อใช้ Power BI Gateway-Personal

> [!NOTE]
> เวอร์ชันปัจจุบันของเกตเวย์สำหรับใช้งานส่วนบุคคลคือ **เกตเวย์ข้อมูลภายในองค์กร (ส่วนบุคคล)** โปรดอัปเดตการติดตั้งเมื่อต้องใช้เวอร์ชันนั้น
> 
> 

## <a name="update-to-the-latest-version"></a>อัปเดตเป็นเวอร์ชันล่าสุด
ปัญหาจำนวนมากสามารถเกิดเมื่อเวอร์ชันเกตเวย์ล้าสมัย  หลักปฏิบัติทั่วไปดีเพื่อให้แน่ใจว่า คุณอยู่บนเวอร์ชันล่าสุดได้ ถ้าคุณยังไม่ได้รับการปรับปรุงเกตเวย์ สำหรับ เดือน หรือนานกว่า พิจารณาติดตั้งเวอร์ชันล่าสุดของเกตเวย์ จากนั้น เห็นถ้าคุณสามารถทำซ้ำปัญหา

## <a name="installation"></a>การติดตั้ง
**เกตเวย์ส่วนบุคคลคือ 64 บิต**-ถ้าเครื่องของคุณเป็นแบบ 32 บิต คุณไม่สามารถติดตั้งเกตเวย์ส่วนบุคคลได้ ระบบปฏิบัติการของคุณจะต้องเป็นเวอร์ชัน 64 บิต ติดตั้ง Windows เวอร์ชัน 64 บิต หรือติดตั้งเกตเวย์ส่วนบุคคลบนเครื่อง 64 บิต

**เกตเวย์ส่วนบุคคลล้มเหลวในการติดตั้งเป็นบริการถึงแม้ว่าคุณเป็นผู้ดูแลระบบภายในสำหรับคอมพิวเตอร์**-ติดตั้งอาจล้มเหลวถ้าผู้ใช้ในกลุ่มผู้ดูแลระบบท้องถิ่นของคอมพิวเตอร์ แต่นโยบายกลุ่มไม่อนุญาตให้ชื่อผู้ใช้ที่ลงชื่อเข้าใช้เป็นแบบ บริการ ในขณะนี้ รับรองนโยบายกลุ่มช่วยให้ผู้ใช้ลงชื่อเข้าใช้เป็นบริการ เรากำลังแก้ปัญหานี้ [เรียนรู้เพิ่มเติม](https://technet.microsoft.com/library/cc739424.aspx)

**การดำเนินการหมดเวลา**-ข้อความนี้เป็นปกติถ้าคอมพิวเตอร์ (เครื่องจริงหรือ VM) ที่คุณกำลังติดตั้งเกตเวย์ส่วนบุคคลมีตัวประมวลผลหลักเดียวกัน ปิดทุกแอปพลิเคชันและปิดการประมวลผลใด ๆ ที่ไม่จำเป็น และลองติดตั้งอีกครั้ง

**Data Management Gateway หรือ Analysis Services Connector ไม่สามารถติดตั้งบนคอมพิวเตอร์เครื่องเดียวกันเป็นเกตเวย์ส่วนบุคคล**- ถ้าคุณมีการ Analysis Services Connector หรือติดตั้ง Data Management Gateway คุณต้องถอนการติดตั้งตัวเชื่อมต่อ หรือ เกตเวย์ แล้ว ลองติดตั้งเกตเวย์ส่วนบุคคล

> [!NOTE]
> ถ้าคุณพบปัญหาระหว่างการติดตั้ง บันทึกการตั้งค่าสามารถให้ข้อมูลเพื่อช่วยให้คุณสามารถแก้ไขปัญหา สำหรับข้อมูลเพิ่มเติม ดู[บันทึกการตั้งค่า](#SetupLogs)
> 
> 

 **กำหนดค่าพร็อกซี**คุณอาจพบปัญหาเกี่ยวกับการกำหนดค่าเกตเวย์ส่วนบุคคลถ้าสภาพแวดล้อมของคุณจำเป็นต้องใช้พร็อกซีได้ เมื่อต้องการเรียนรู้เพิ่มเติมเกี่ยวกับวิธีการกำหนดค่าพร็อกซีข้อมูล ดู[กำหนดค่าพร็อกซีสำหรับเกตเวย์ Power BI ](service-gateway-proxy.md)

## <a name="schedule-refresh"></a>กำหนดตารางเวลาการรีเฟรช
**ข้อผิดพลาด: ข้อมูลประจำตัวที่จัดเก็บไว้ในระบบคลาวด์สูญหาย**

คุณอาจได้รับข้อผิดพลาดนี้ในการตั้งค่าสำหรับ\<ชุดข้อมูล\>ถ้าคุณมีการรีเฟรชตามกำหนดการแล้วถอนการติดตั้ง และติดตั้งเกตเวย์ส่วนบุคคลได้ เมื่อถอนการติดตั้งเกตเวย์ส่วนบุคคล ข้อมูลประจำตัวแหล่งข้อมูลสำหรับชุดข้อมูลที่ถูกกำหนดค่าสำหรับการรีเฟรชจะถูกเอาออกจากบริการ Power BI

**วิธีแก้ไข:** ใน Power BI ไปที่การตั้งค่าการรีเฟรชสำหรับชุดข้อมูล ในการจัดการแหล่งข้อมูล สำหรับแหล่งข้อมูลใด ๆ ที่มีข้อผิดพลาด เลือก**แก้ไขข้อมูลประจำตัว**และลงชื่อเข้าใช้ไปยังแหล่งข้อมูลอีกครั้ง

**ข้อผิดพลาด: ข้อมูลประจำตัวที่ระบุไว้สำหรับชุดข้อมูลไม่ถูกต้อง โปรดอัปเดตข้อมูลประจำตัวผ่านการรีเฟรชหรือในกล่องโต้ตอบการตั้งค่าแหล่งข้อมูลเพื่อดำเนินการต่อ**

**โซลูชัน**: ถ้าคุณได้รับข้อความข้อมูลประจำตัว อาจหมายถึง:

* ตรวจสอบให้แน่ใจว่า ชื่อผู้ใช้และรหัสผ่านสำหรับการลงชื่อเข้าใช้แหล่งข้อมูลมีการปรับปรุง ใน Power BI ไปที่การตั้งค่ารีเฟรชสำหรับชุดข้อมูลนั้น ในการจัดการแหล่งข้อมูล เลือก**แก้ไขข้อมูลประจำตัว**เพื่อปรับปรุงข้อมูลประจำตัวสำหรับแหล่งข้อมูล
* Mashups ระหว่างแหล่งข้อมูล cloud และภายในองค์กรแหล่ง ในแบบสอบถามเดียว ล้มเหลวในการรีเฟรชในเกตเวย์ส่วนบุคคลถ้าหนึ่งในแหล่งข้อมูลใช้ OAuth สำหรับการรับรองความถูกต้อง ตัวอย่างของปัญหานี้คือ mashup ระหว่าง CRM Online และเป็น SQL Server ภายในเครื่อง Mashup ล้มเหลวเนื่องจาก CRM Online ต้องมี OAuth
  
  ข้อผิดพลาดนี้เป็นปัญหาทราบแล้ว และจะถูกค้นหาที่ เมื่อต้องแก้ปัญหา มีแบบสอบถามแยกต่างหากสำหรับแหล่งข้อมูล cloud และแหล่งข้อมูลภายในองค์กร แล้ว ใช้การผสาน หรือผนวกคิวรีเพื่อรวมเข้าด้วยกัน

**ข้อผิดพลาด: แหล่งข้อมูลที่ไม่สนับสนุน**

**วิธีแก้ไข:** ถ้าคุณได้รับข้อความแหล่งข้อมูลที่ไม่สนับสนุนในการตั้งค่าการกำหนดเวลารีเฟรช อาจหมายถึง: 

* แหล่งข้อมูลไม่รองรับสำหรับการรีเฟรชใน Power BI ในขณะนี้ 
* สมุดงาน Excel ไม่ประกอบด้วยแบบจำลองข้อมูล แผ่นข้อมูล Power BI ในขณะนี้สนับสนุนเฉพาะการรีเฟรชถ้าสมุดงาน Excel ที่อัปโหลดประกอบด้วยแบบจำลองข้อมูล เมื่อคุณนำเข้าข้อมูลโดยใช้ Power Query ใน Excel ตรวจสอบให้แน่ใจว่าได้เลือกตัวเลือกที่จะโหลดข้อมูลไปยังโมเดลข้อมูล ตัวเลือกนี้ทำให้แน่ใจจะนำเข้าข้อมูลลงในแบบจำลองข้อมูล 

**ข้อผิดพลาด: [ไม่สามารถรวมข้อมูล] &lt;ส่วนแบบสอบถาม&gt;/&lt;... &gt; / &lt;... &gt;กำลังเข้าถึงแหล่งข้อมูลที่มีระดับความเป็นส่วนตัว ที่ไม่สามารถใช้ร่วมกัน โปรดสร้างชุดรวมข้อมูลนี้อีกครั้ง**

**โซลูชัน**: ข้อผิดพลาดนี้เนื่องจากข้อจำกัดที่ระดับความเป็นส่วนตัวและชนิดของแหล่งข้อมูลที่คุณกำลังใช้อยู่

**ข้อผิดพลาด: แหล่งที่มาของข้องมูลเกิดข้อผิดพลาด: เราไม่สามารถแปลงค่า "\[ตาราง\]" เป็นพิมพ์ตารางได้**

**โซลูชัน**: ข้อผิดพลาดนี้เนื่องจากข้อจำกัดที่ระดับความเป็นส่วนตัวและชนิดของแหล่งข้อมูลที่คุณกำลังใช้อยู่

**ข้อผิดพลาด: มีเนื้อที่ไม่เพียงพอสำหรับแถวนี้**

ข้อผิดพลาดนี้เกิดขึ้นถ้าคุณมีแถวมากกว่า 4 MB เดียวในขนาด ค้นหาแถวจากแหล่งข้อมูลของคุณ และพยายามทำการกรองออก หรือลดขนาดสำหรับแถวนั้น

## <a name="data-sources"></a>แหล่งข้อมูล
**ผู้ให้บริการข้อมูลหายไป**– เกตเวย์ส่วนบุคคลเป็นเวอร์ชัน 64 บิตเท่านั้น ต้องใช้ตัวให้บริการข้อมูลเวอร์ชัน 64 บิตในการติดตั้งบนคอมพิวเตอร์เครื่องเดียวกันกับที่ติดตั้งเกตเวย์ส่วนบุคคล ตัวอย่างเช่น ถ้าแหล่งข้อมูลในชุดข้อมูลเป็น Microsoft Access คุณต้องติดตั้งตัวให้บริการ 64 บิต ACE บนคอมพิวเตอร์เครื่องเดียวกันกับที่คุณติดตั้งเกตเวย์ส่วนบุคคล  

>[!NOTE]
>ถ้าคุณมีเวอร์ชัน 32 บิต Excel คุณไม่สามารถติดตั้งผู้ให้บริการ ACE เวอร์ชัน 64 บิตบนคอมพิวเตอร์เครื่องเดียวกัน

**ไม่สนับสนุนการรับรองความถูกต้องของ Windows สำหรับฐานข้อมูล Access** ในขณะนี้ Power BI สนับสนุนเฉพาะแบบไม่ระบุชื่อสำหรับฐานข้อมูล Access เรากำลังทำงานบนการเปิดใช้งานการรับรองความถูกต้องของ Windows สำหรับฐานข้อมูล Access

**ลงชื่อเข้าใช้ในข้อผิดพลาดเมื่อใส่ข้อมูลประจำตัวสำหรับแหล่งข้อมูล**-ถ้าคุณได้รับข้อผิดพลาดนี้เมื่อใส่ข้อมูลประจำตัวของ Windows สำหรับแหล่งข้อมูล คุณอาจยังคงอยู่บนเกตเวย์ส่วนบุคคลเวอร์ชันเก่ากว่าได้ [ติดตั้งเวอร์ชันล่าสุดของเกตเวย์ Power BI - ส่วนบุคคล](https://powerbi.microsoft.com/gateway/)

  ![](media/service-admin-troubleshooting-power-bi-personal-gateway/pbi_pg_credentialserror.jpg.png)

**ข้อผิดพลาด: ข้อผิดพลาดการลงชื่อเข้าใช้เมื่อเลือกการรับรองความถูกต้องของ Windows สำหรับแหล่งข้อมูลโดยใช้ ACE OLEDB** ถ้าคุณได้รับข้อความข้อผิดพลาดต่อไปนี้เมื่อใส่ข้อมูลประจำตัวของแหล่งข้อมูลสำหรับแหล่งข้อมูลโดยใช้ผู้ให้บริการ ACE OLEDB:

![](media/service-admin-troubleshooting-power-bi-personal-gateway/aceoledberror.png)

Power BI ไม่อยู่ในขณะนี้สนับสนุนการรับรองความถูกต้องของ Windows สำหรับแหล่งข้อมูลโดยใช้ผู้ให้บริการ ACE OLEDB

**วิธีแก้ไข:** เมื่อต้องหลีกเลี่ยงข้อผิดพลาดนี้ คุณสามารถเลือก**รับรองความถูกต้องแบบไม่ระบุชื่อ**ได้ สำหรับผู้ให้บริการ ACE OLEDB แบบดั้งเดิม ข้อมูลประจำตัวแบบไม่ระบุชื่อไม่เท่ากับข้อมูลประจำตัวของ Windows

## <a name="tools-for-troubleshooting"></a>เครื่องมือสำหรับการแก้ไขปัญหา
### <a name="refresh-history"></a>ประวัติการรีเฟรช
**ประวัติการรีเฟรช**ช่วยให้คุณเห็นข้อผิดพลาดเกิดขึ้น และให้ข้อมูลที่เป็นประโยชน์ถ้าคุณต้องการสร้างคำขอการสนับสนุน คุณสามารถดูทั้งจัดกำหนดการ และ ตาม รีเฟรช นี่คือวิธีที่คุณจะไป**ประวัติการรีเฟรช**

1. ในพื้นที่นำทางของ Power BI ใน**ชุดข้อมูล** เลือกชุดข้อมูล &gt;เปิดเมนู&gt; **กำหนดตารางเวลาการรีเฟรช**
   ![](media/service-admin-troubleshooting-power-bi-personal-gateway/scheduled-refresh.png)
1. ใน**ตั้งค่าสำหรับ...** เลือก**ประวัติการรีเฟรช**  
   ![](media/service-admin-troubleshooting-power-bi-personal-gateway/scheduled-refresh-2.png)
   
   ![](media/service-admin-troubleshooting-power-bi-personal-gateway/refresh-history.png)

### <a name="event-logs"></a>ไฟล์บันทึกเหตุการณ์
บันทึกเหตุการณ์ต่าง ๆ สามารถให้ข้อมูล สอง**Data Management Gateway**และ**PowerBIGateway**อยู่ถ้าคุณเป็นผู้ดูแลระบบบนเครื่อง  ถ้าคุณไม่ใช่ผู้ดูแลระบบ และคุณกำลังใช้เกตเวย์ส่วนบุคคล คุณจะเห็นรายการบันทึกภายในการ**แอปพลิเคชัน**บันทึก

บันทึก**Data Management Gateway**และ**PowerBIGateway**อยู่ใต้**บันทึกแอปพลิเคชันและบริการ**

![](media/service-admin-troubleshooting-power-bi-personal-gateway/event-logs.png)

### <a name="fiddler-trace"></a>ติดตาม Fiddler
[Fiddler](http://www.telerik.com/fiddler) เป็นเครื่องมือฟรีจาก Telerik ที่ใช้ตรวจดูการรับส่งข้อมูลใน HTTP คุณสามารถดูการติดต่อสื่อสารกับบริการ Power BI จากเครื่องของลูกค้า สื่อสารนี้อาจแสดงข้อผิดพลาดและข้อมูลอื่น ๆ ที่เกี่ยวข้อง

![](media/service-admin-troubleshooting-power-bi-personal-gateway/fiddler.png)

### <a name="setup-logs"></a>บันทึกการตั้งค่า
ถ้าตัว**เกตเวย์ส่วนบุคคล**ล้มเหลวในการติดตั้ง คุณจะเห็นลิงก์เพื่อแสดงบันทึกการตั้งค่า บันทึกการตั้งค่าสามารถแสดงรายละเอียดเกี่ยวกับความล้มเหลว บันทึกเหล่านี้จะติดตั้ง Windows บันทึก หรือที่ เรียกว่าบันทึก MSI บันทึกเหล่านี้อาจค่อนข้างซับซ้อนและยากต่อการอ่าน โดยทั่วไปแล้ว ข้อผิดพลาดเป็นผลลัพธ์อยู่ที่ด้านล่าง แต่ในการระบุสาเหตุของข้อผิดพลาดไม่สำคัญ ไม่สามารถเป็นผลของข้อผิดพลาดในบันทึกอื่น หรือเป็นผลของข้อผิดพลาดที่สูงขึ้นในบันทึกดังกล่าว

![](media/service-admin-troubleshooting-power-bi-personal-gateway/setup-log.png)

หรือ คุณสามารถไปของคุณ**โฟลเดอร์ Temp** (% temp %) และค้นหาไฟล์ที่ขึ้นต้นด้วย**Power\_BI\_**

> [!NOTE]
> การไปที่ %temp% อาจนำคุณไปยังโฟลเดอร์ย่อยของโฟลเดอร์ชั่วคราว การ**Power\_BI\_** ไฟล์อยู่ในรากของไดเรกทอรีชั่วคราว  คุณอาจจำเป็นต้องไปยังหนึ่งระดับหรือสองระดับ
>
