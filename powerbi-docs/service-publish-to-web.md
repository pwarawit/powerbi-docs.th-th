# <a name="publish-to-web-from-power-bi"></a>เผยแพร่บนเว็บจาก Power BI

ด้วย Power BI **เผยแพร่ไปยังเว็บ**ตัวเลือก คุณสามารถฝังได้อย่างง่ายดายแสดงภาพ Power BI แบบโต้ตอบแบบออนไลน์ เช่นโพสต์ในบล็อก เว็บไซต์ ผ่านอีเมลหรือ social media จากอุปกรณ์ใดก็ได้ คุณสามารถแก้ไข อัปเดต รีเฟรช หรือยกเลิกการแชร์การแสดงภาพที่คุณเผยแพร่

> [!WARNING]
> เมื่อคุณใช้**เผยแพร่ไปยังเว็บ**ทุกคนบนอินเทอร์เน็ตสามารถดูรายงานที่เผยแพร่หรือรูปภาพของคุณได้ ซึ่งจำเป็นต้องมีการรับรองความถูกต้อง และรวมถึงดูข้อมูลระดับรายละเอียดรายงานของคุณรวม ก่อนที่จะเผยแพร่รายงาน ตรวจสอบให้แน่ใจว่า จะให้คุณสามารถแชร์ข้อมูลและการแสดงภาพแบบสาธารณะ อย่าเผยแพร่ข้อมูลที่เป็นความลับหรือมีกรรมสิทธิ์ ถ้ามีข้อสงสัย ให้ตรวจสอบนโยบายขององค์กรของคุณก่อนเผยแพร่

>[!Note]
>ในการฝังเนื้อหาของคุณได้อย่างปลอดภัยในพอร์ทัลภายในหรือเว็บไซต์ ให้ใช้ตัวเลือกการ[ฝัง](service-embed-secure.md)หรือ[ฝังใน SharePoint Online](service-embed-report-spo.md) ซึ่งรับรองสิทธิ์ทั้งหมด และความปลอดภัยของข้อมูลที่บังคับใช้เมื่อผู้ใช้ของคุณกำลังดูข้อมูลภายในของคุณ

## <a name="how-to-use-publish-to-web"></a>วิธีใช้เผยแพร่ไปยังเว็บ

**เผยแพร่ไปยังเว็บ**จะพร้อมใช้งานสำหรับรายงานในส่วนบุคคล หรือกลุ่มพื้นที่ทำงานของคุณสามารถแก้ไข  ไม่พร้อมใช้งานสำหรับรายงานที่แชร์กับคุณ หรือรูปภาพพึ่งพาความปลอดภัยระดับแถวเพื่อรักษาความปลอดภัยข้อมูล ดู[ **ข้อจำกัด** ](#limitations)ด้านล่างสำหรับรายชื่อทั้งหมดของกรณีที่**เผยแพร่ไปยังเว็บ**ไม่รองรับ ตรวจทานการ**เตือน**ก่อนหน้าในบทความนี้ก่อนที่ใช้**เผยแพร่ไปยังเว็บ**

ต่อไปนี้*วิดีโอสั้น ๆ*แสดงวิธีการทำงานของฟีเจอร์นี้ แล้ว ลองทำด้วยตนเองในขั้นตอนด้านล่าง

<iframe width="560" height="315" src="https://www.youtube.com/embed/UF9QtqE7s4Y" frameborder="0" allowfullscreen></iframe>

ขั้นตอนต่อไปนี้อธิบายวิธีการใช้**เผยแพร่ไปยังเว็บ**

1. เปิดรายงานในพื้นที่ทำงานของคุณที่คุณสามารถแก้ไข และเลือก**ไฟล์ > เผยแพร่ไปยังเว็บ**

   ![PtW1](media/service-publish-to-web/publish_to_web1.png)

2. ตรวจทานเนื้อหา และเลือกกล่องโต้ตอบ**สร้างโค้ดฝังตัว**

   ![PtW2](media/service-publish-to-web/publish_to_web2_ga.png)

3. ตรวจทานคำเตือน ตามที่แสดงที่นี่ และยืนยันว่า ข้อมูลสามารถฝังในเว็บไซต์สาธารณะ ถ้า เลือก**เผยแพร่**

   ![PtW3](media/service-publish-to-web/publish_to_web3_ga.png)

4. กล่องโต้ตอบปรากฏขึ้นพร้อมลิงก์ คุณสามารถส่งลิงก์นี้ในอีเมล ฝังในรหัสเช่น iFrame หรือวางโดยตรงลงในเว็บเพจหรือบล็อก

   ![PtW4](media/service-publish-to-web/publish_to_web4.png)

5. ถ้าคุณเคยสร้างโค้ดฝังตัวสำหรับรายงาน และคุณเลือก**เผยแพร่ไปยังเว็บ**คุณจะไม่เห็นกล่องโต้ตอบในขั้นตอนที่ 2-4 แทน**โค้ดฝังตัว**โต้ตอบปรากฏขึ้น:

   ![PtW5](media/service-publish-to-web/publish_to_web5.png)

   คุณสามารถสร้างโค้ดฝังตัวหนึ่งโค้ดสำหรับแต่ละรายงาน


## <a name="tips-and-tricks-for-view-modes"></a>คำแนะนำ และเคล็ดลับสำหรับโหมดมุมมอง

เมื่อคุณฝังเนื้อหาภายในโพสต์ในบล็อก คุณโดยทั่วไปแล้วจำเป็นต้องให้พอดีภายในขนาดหน้าจอที่เฉพาะเจาะจง  คุณสามารถปรับความสูงและความกว้างเป็นแท็ก iFrame ตามความจำเป็น อย่างไรก็ตาม คุณจำเป็นต้องให้แน่ใจว่า รายงานของคุณพอดีกับพื้นที่ที่ระบุของ iFrame ดังนั้นคุณยังต้องตั้งค่าเป็นโหมดมุมมองที่เหมาะสมเมื่อแก้ไขรายงาน

ตารางต่อไปนี้มีคำแนะนำเกี่ยวกับโหมดมุมมอง และวิธีจะปรากฏเมื่อถูกฝังตัว

## <a name="manage-embed-codes"></a>จัดการโค้ดแบบฝังตัว

เมื่อคุณสร้างการ**เผยแพร่ไปยังเว็บ**โค้ดฝังตัว คุณสามารถจัดการรหัสของคุณจากการ**ตั้งค่า**เมนูใน Power BI การจัดการโค้ดแบบฝังตัวมีความสามารถในการลบภาพปลายทางหรือรายงานสำหรับรหัส (แสดงโค้ดฝังตัวไม่สามารถใช้งาน), หรือการรับโค้ดฝังตัว

1. เพื่อจัดการฝังรหัส**เผยแพร่ไปยังเว็บ**ของคุณ เปิดเฟือง**การตั้งค่า** แล้วเลือก**จัดการโค้ดฝังตัว**

   ![PtW8](media/service-publish-to-web/publish_to_web8.png)

2. โค้ดแบบฝังตัวของคุณปรากฏขึ้น

   ![PtW9](media/service-publish-to-web/publish_to_web9.png)

3. คุณสามารถเรียกใช้ หรือลบโค้ดฝังตัว ลบลิงก์ใด ๆ ไปยังรายงานหรือรูปภาพที่ปิดใช้งาน

   ![PtW10](media/service-publish-to-web/publish_to_web10.png)

4. ถ้าคุณเลือก**ลบ**คุณกำลังขอการยืนยัน

   ![PtW11](media/service-publish-to-web/publish_to_web11.png)

## <a name="updates-to-reports-and-data-refresh"></a>อัปเดตรายงานและรีเฟรชข้อมูล

หลังจากที่คุณสร้างของคุณ**เผยแพร่ไปยังเว็บ**โค้ดฝังตัวและแชร์ รายงานอัปเด มีการเปลี่ยนแปลงใด ๆ ที่คุณทำ และฝังตัวลิงก์โค้ดเป็นเปิดใช้งานทันที และทุกคนที่เปิดลิงก์สามารถดู หลังจากการดำเนินการนี้เริ่มต้น อย่างไรก็ตาม ปรับปรุงการรายงานหรือภาพอาจใช้เวลาประมาณหนึ่งชั่วโมงก่อนที่จะสามารถมองเห็นผู้ใช้ของคุณ ถ้าคุณต้องการอัปเดตความสามารถใช้งานแบบทันทีของคุณ คุณสามารถลบโค้ดแบบฝังตัวและสร้างขึ้นใหม่ เมื่อต้องการเรียนรู้เพิ่มเติม ดู[ **วิธีการทำงาน** ](#howitworks)ส่วนในภายหลังในบทความนี้ 

## <a name="data-refresh"></a>การรีเฟรชข้อมูล

รีเฟรชข้อมูลเป็นภาพสะท้อนในรายงานแบบฝังตัวหรือแบบรูปภาพอัตโนมัติของคุณ อาจใช้เวลาประมาณ 1 ชั่วโมงสำหรับรีเฟรชข้อมูลเพื่อให้สามารถมองเห็นได้จากโค้ดแบบฝังตัว เมื่อต้องการปิดใช้งานการรีเฟรชโดยอัตโนมัติ คุณสามารถเลือก**ไม่ได้รีเฟรช**ตามกำหนดการสำหรับชุดข้อมูลใช้รายงานได้  

## <a name="custom-visuals"></a>ภาพแบบกำหนดเอง

ภาพแบบกำหนดเองรองรับ**การเผยแพร่ไปยังเว็บ** เมื่อคุณใช้**เผยแพร่ไปยังเว็บ**ผู้ใช้ที่คุณแชร์ภาพที่เผยแพร่ของคุณไม่จำเป็นต้องเปิดใช้งานวิชวลแบบกำหนดเองเพื่อดูรายงาน

## <a name="limitations"></a>ข้อจำกัด

**เผยแพร่ไปยังเว็บ**ได้รับการสนับสนุนสำหรับแหล่งข้อมูลส่วนใหญ่ของข้อมูล และรายงานในบริการ Power BI อย่างไรก็ตาม ต่อไปนี้จะ**ไม่อยู่ในขณะนี้ได้รับการสนับสนุน หรือพร้อมใช้งาน**กับ**เผยแพร่ไปยังเว็บ** :

- รายงานการใช้ความปลอดภัยระดับแถว
- รายงานที่ใช้แหล่งข้อมูลแบบไลฟ์ใดๆ รวมถึง Analysis Services Tabular ที่โฮสต์ภายในองค์กร Analysis Services Multidimensional และ Azure Analysis Services
- รายงานทีแชร์กับคุณโดยตรง หรือผ่าแพ็คเนื้อหาขององค์กร
- รายงานในกลุ่มที่คุณไม่ใช่สมาชิกที่มีสิทธิ์แก้ไข
- วิชวล " R " จะไม่สนับสนุนใน**เผยแพร่ไปยังเว็บ**รายงาน
- ส่งออกข้อมูลจากวิชวลในรายงาน ซึ่งมีการเผยแพร่ไปยังเว็บ
- ArcGIS Maps for Power BI วิชวล
- รายงานที่ประกอบด้วยหน่วยวัด DAX ระดับรายงาน
- ลงชื่อเข้าระบบข้อมูลเดียวคิวรีแบบจำลอง
- [รักษาความปลอดภัยของข้อมูลที่เป็นความลับ หรือกรรมสิทธิ์](#publish-to-web-from-power-bi)
- ความสามารถในการรับรองความถูกต้องโดยอัตโนมัติที่มาพร้อมกับการ**ฝัง**ตัวเลือกที่ไม่ทำงานกับ Power BI JavaScript API สำหรับ Power BI JavaScript API ให้ใช้[ผู้ใช้เป็นเจ้าของข้อมูล](developer/embed-sample-for-your-organization.md)ในการฝัง เรียนรู้เพิ่มเติมเกี่ยวกับ[ผู้ใช้เป็นเจ้าของข้อมูล](developer/embed-sample-for-your-organization.md)

## <a name="tenant-setting"></a>การตั้งค่าผู้เช่า

ผู้ดูแลระบบ power BI สามารถเปิด หรือปิดการ**เผยแพร่ไปยังเว็บ**คุณลักษณะได้ พวกเขายังสามารถจำกัดการเข้าถึงกลุ่มที่เฉพาะเจาะจง ซึ่งสามารถส่งผลกระทบต่อความสามารถในการสร้างโค้ดฝังตัว

## <a name="understanding-the-embed-code-status-column"></a>ทำความเข้าใจเกี่ยวกับคอลัมน์สถานะโค้ดฝังตัว

การ**โค้ดฝังตัวจัดการ**หน้าประกอบด้วยคอลัมน์สถานะ ตามค่าเริ่มต้น โค้ดฝังตัวจะ**Active**แต่อาจเป็นหนึ่งในสถานะที่แสดงอยู่ด้านล่าง

| สถานะ | คำอธิบาย |
| --- | --- |
| **ใช้งานอยู่** |รายงานจะพร้อมใช้งานสำหรับผู้ใช้อินเทอร์เน็ตเพื่อดูและโต้ตอบ |
| **ถูกบล็อก** |เนื้อหารายงานการละเมิดการ[Power BI เงื่อนไขการใช้บริการ](https://powerbi.microsoft.com/terms-of-service) Microsoft ได้บล็อกไว้ ติดต่อฝ่ายสนับสนุนหากคุณเชื่อว่าเนื้อหาที่ถูกบล็อกมาจากความผิดพลาด |
| **ไม่รองรับ** |ชุดข้อมูลของรายงานใช้ความปลอดภัยระดับแถว หรือไม่รองรับการกำหนดค่าอื่น ดู[ **ข้อจำกัด** ](#limitations)ส่วนสำหรับรายการทั้งหมด |
| **ถูกละเมิด** |โค้ดฝังอยู่ภายนอกผู้เช่ากำหนดนโยบาย ซึ่งมักเกิดขึ้นเมื่อสร้างโค้ดฝังตัว จากนั้นใน**เผยแพร่ไปยังเว็บ**ตั้งค่าผู้เช่าถูกเปลี่ยนเพื่อไม่รวมผู้ใช้ที่เป็นเจ้าของโค้ดฝังตัว ถ้าตั้งค่าผู้เช่าถูกปิดใช้งาน หรือผู้ใช้ไม่ได้รับอนุญาตให้สร้าง โค้ดฝังตัว ที่มีอยู่แสดงรหัสฝังแอ**ถูกละเมิด**สถานะ |

## <a name="how-to-report-a-concern-with-publish-to-web-content"></a>วิธีการรายงานข้อจำกัดเกี่ยวกับการเผยแพร่ไปยังเนื้อหาบนเว็บ

เมื่อต้องการรายงานข้อจำกัดที่เกี่ยวข้องกับ**เผยแพร่ไปยังเว็บ**เนื้อหาที่ฝังในเว็บไซต์หรือบล็อก ใช้การ**สถานะ**ไอคอนในแถบด้านล่าง ดังที่แสดงในรูปต่อไปนี้ คุณจะถูกขอให้ส่งอีเมลไปยังไมโครซอฟท์เพื่ออธิบายกังวลของคุณ Microsoft จะประเมินเนื้อหาที่ยึดตามเงื่อนไขของบริการของ Power BI และดำเนินการที่เหมาะสม

เมื่อต้องรายงานข้อจำกัด เลือกแบบ**สถานะ**ไอคอนในแถบด้านล่างของการ**เผยแพร่ไปยังเว็บ**รายงานคุณเห็น

<a name="howitworks"></a>
## <a name="how-it-works-technical-details"></a>ทำงานอย่างไร (รายละเอียดทางเทคนิค)

เมื่อคุณสร้างการใช้ฝังรหัส**เผยแพร่ไปยังเว็บ**รายงานจะทำให้ผู้ใช้อินเทอร์เน็ตมองเห็น ซึ่งเป็นสาธารณชน ดังนั้นคุณสามารถคาดหวังผู้ชมแชร์รายงานผ่านสื่อทางสังคมในอนาคตได้อย่างง่ายดาย ขณะที่ผู้ใช้ดูรายงาน โดยเปิดผ่าน URL สาธารณะโดยตรงหรือดูแบบฝังในเว็บเพจหรือบล็อก Power BI เก็บการให้คำนิยามรายงานและผลลัพธ์ของคิวรี่ที่จำเป็นต้องดูรายงาน ซึ่งทำให้แน่ใจว่า ผู้ใช้พร้อมกันสามารถดูรายงานโดยไม่ส่งผลกระทบต่อประสิทธิภาพการทำงาน

แคชเป็นแบบ long-lived ดังนั้นถ้าคุณปรับปรุงข้อกำหนดรายงาน (ตัวอย่าง ถ้าคุณเปลี่ยนโหมดมุมมอง) หรือรีเฟรชข้อมูลรายงาน อาจใช้เวลาประมาณหนึ่งชั่วโมงก่อนที่เปลี่ยนแปลงจะปรากฏในเวอร์ชันของรายงานดูผู้ใช้ของคุณ จึงขอแนะนำให้คุณเริ่มทำงานก่อน และสร้างโค้ดฝังตัว**เผยแพร่ไปยังเว็บ** เมื่อเพียงคุณพอใจกับการตั้งค่า
