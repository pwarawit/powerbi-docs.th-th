# <a name="tutorial-build-a-machine-learning-model-in-power-bi-preview"></a>บทช่วยสอน: สร้างแบบจำลอง Machine Learning ใน Power BI (ตัวอย่าง)

ในบทความบทช่วยสอน คุณใช้**อัตโนมัติ Machine Learning**เพื่อสร้าง และปรับใช้แบบจำลองการคาดการณ์ไบนารีใน Power BI บทช่วยสอนมีคำแนะนำสำหรับการสร้างแบบ dataflow Power BI และใช้เอนทิตีกำหนดไว้ใน dataflow การฝึกอบรม และการตรวจสอบเครื่องเรียนรู้แบบจำลองได้โดยตรงใน Power BI เราใช้รูปแบบนั้นสำหรับการให้คะแนนในการสร้างการคาดการณ์

ก่อนอื่น คุณจะสร้างเครื่องการคาดการณ์ไบนารีเรียนรู้แบบจำลอง เพื่อทำนายจุดประสงค์ซื้อของ shoppers ออนไลน์ที่ยึดตามชุดของแอตทริบิวต์ของเซสชันแบบออนไลน์ ใช้ชุดข้อมูลเรียนรู้เครื่อง benchmark สำหรับแบบทดสอบนี้ เมื่อได้รับแบบจำลองการฝึกฝน Power BI จะสร้างรายงานตรวจสอบอธิบายผลลัพธ์แบบจำลองโดยอัตโนมัติ คุณสามารถตรวจสอบรายงานการตรวจสอบ และนำไปใช้แบบจำลองข้อมูลของคุณสำหรับการให้คะแนน

บทช่วยสอนนี้ประกอบด้วยขั้นตอนต่อไปนี้:

> [!div class="checklist"]
> * สร้าง dataflow ที่ มีข้อมูลป้อนเข้า
> * สร้าง และฝึกแบบจำลองการเรียนรู้เครื่อง
> * ตรวจสอบรายงานการตรวจสอบแบบจำลอง
> * นำรูปแบบไปใช้กับเอนทิตี dataflow
> * ใช้ผลลัพธ์ได้คะแนนจากแบบจำลองในรายงาน Power BI

## <a name="create-a-dataflow-with-the-input-data"></a>สร้าง dataflow ที่ มีข้อมูลป้อนเข้า

ส่วนแรกของบทช่วยสอนนี้คือการ สร้าง dataflow ที่ มีข้อมูลป้อนเข้า ขั้นตอนที่ใช้เวลาขั้นตอน ดังที่แสดงในส่วนต่อไปนี้ เริ่มต้น ด้วยการรับข้อมูล

### <a name="get-data"></a>รับข้อมูล

ขั้นตอนแรกในการสร้าง dataflow จะมีแหล่งข้อมูลของคุณพร้อมแล้ว ในกรณีของเรา เราใช้ชุดข้อมูลเรียนรู้เกี่ยวกับเครื่องจากชุดของเซสชันแบบออนไลน์ บาง culminated ในการซื้อ ชุดข้อมูลประกอบด้วยชุดของแอตทริบิวต์เกี่ยวกับเซสชันเหล่านี้ ซึ่งเราจะใช้สำหรับการฝึกอบรมแบบจำลองของเรา

คุณสามารถดาวน์โหลดชุดข้อมูลจากเว็บไซต์ UC Irvine  เรายังมีนี้พร้อมใช้งาน สำหรับวัตถุประสงค์ของบทช่วยสอนนี้ จากลิงก์ต่อไปนี้: [online_shoppers_intention.csv](https://raw.githubusercontent.com/santoshc1/PowerBI-AI-samples/master/Tutorial_AutomatedML/online_shoppers_intention.csv)

### <a name="create-the-entities"></a>สร้างเอนทิตี

หากต้องสร้างเอนทิตีในกระแสข้อมูลของคุณ ให้ลงชื่อเข้าใช้บริการของ Power BI และนำทางไปยังพื้นที่ทำงานบนความจุเฉพาะของคุณที่มีตัวอย่าง AI เปิดการใช้งานอยู่

ถ้าคุณไม่มีพื้นที่ทำงาน คุณสามารถสร้างได้ โดยการเลือก**พื้นที่ทำงาน**ในเมนูการนำทางด้านซ้ายในบริการ Power BI และเลือก**สร้างพื้นที่ทำงานแอ**ที่ด้านล่างของบานหน้าต่างที่ ปรากฏขึ้น ซึ่งเปิดบานหน้าต่างด้านขวาให้ใส่รายละเอียดพื้นที่ทำงาน ใส่ชื่อพื้นที่ทำงาน และเลือก**ขั้นสูง** ยืนยันว่า พื้นที่ทำงานที่ใช้โดยใช้ปุ่มตัวเลือกความจุเฉพาะ และจะถูกกำหนดให้กับอินสแตนซ์ที่มีความจุที่มีตัวอย่าง AI ที่เปิดใช้งาน จากนั้นเลือก**บันทึก**

![สร้างพื้นที่ทำงาน](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-01.png)

เมื่อมีสร้างพื้นที่ทำงาน คุณสามารถเลือก**ข้าม**ในด้านล่างขวาของหน้าจอยินดีต้อนรับ ดังที่แสดงในรูปต่อไปนี้ได้

![ข้ามได้ถ้าคุณมีพื้นที่ทำงาน](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-02.png)

เลือกคำ**Dataflows (ตัวอย่าง)** แท็บ เลือกคำ**สร้าง**ปุ่มที่ด้านบนขวาของพื้นที่ทำงาน และจากนั้นเลือก**Dataflow**

![สร้าง dataflow](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-03.png)

เลือก **เอนทิตีใหม่** ซึ่งเปิดใช้งานการ**Power Query**ตัวแก้ไขในเบราว์เซอร์

![เพิ่มเอนทิตีใหม่](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-04.png)

เลือก**ไฟล์ข้อ ความ/CSV**เป็นแหล่งข้อมูล แสดงในรูปต่อไปนี้

![ไฟล์ข้อ ความ/CSF ที่เลือก](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-05.png)

ในการ**เชื่อมต่อกับแหล่งข้อมูล**ที่ปรากฏขึ้นถัดไป วางลิงก์ต่อไปนี้ไป*online_shoppers_intention.csv*ลงใน**พาธของไฟล์หรือ URL**กล่อง จากนั้น **ถัดไป**

`https://raw.githubusercontent.com/santoshc1/PowerBI-AI-samples/master/Tutorial_AutomatedML/online_shoppers_intention.csv`

![พาธของไฟล์](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-06.png)

ตัวแก้ไข Power Query แสดงตัวอย่างของข้อมูลจากไฟล์ CSV เลือก**แปลงตาราง**ในคำสั่ง ribbon แล้วเลือก**ใช้แถวแรกเป็นส่วนหัวของ**จากเมนูที่ปรากฏขึ้น ซึ่งเพิ่มการ_ส่วนหัวที่เลื่อนระดับ_ขั้นตอนคิวรีลงในการ**ใช้ขั้นตอน**ส่วนทางด้านขวาของหน้าจอ คุณสามารถเปลี่ยนชื่อคิวรีให้เป็นชื่อที่เข้าใจง่าย โดยการเปลี่ยนค่าในการ**ชื่อ**พบกล่องในบานหน้าต่างด้านขวาได้ ตัวอย่าง คุณสามารถเปลี่ยนชื่อคิวรีเพื่อ_เยี่ยมชม Online_ได้

![เปลี่ยนเป็นชื่อที่เรียกง่าย](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-07.png)

ชนิดแอตทริบิวต์ข้อมูลในชุดข้อมูลนี้บาง_ตัวเลข_หรือ_บูลีน_แม้ ว่าสิ่งเหล่านี้อาจถูกแปลเป็นสตริงโดย**Power Query** เลือกไอคอนชนิดแอตทริบิวต์ที่ด้านบนของแต่ละส่วนหัวของคอลัมน์เพื่อเปลี่ยนคอลัมน์รายการด้านล่างเพื่อชนิดต่อไปนี้:

* **เลขทศนิยม:** Administrative_Duration Informational_Duration ProductRelated_Duration BounceRates ExitRates PageValues SpecialDay
* **True/False:** วันหยุดสุดสัปดาห์ รายได้

![เปลี่ยนชนิดข้อมูล](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-08.png)

การ**คาดการณ์ไบนารี**แบบจำลองที่เราจะฝึกจำเป็นต้องมีเขตข้อมูลบูลีนเป็นป้ายชื่อที่ระบุผลลัพธ์จากการสังเกตที่ผ่านมา ในชุดข้อมูลนี้ การ_รายได้_แอตทริบิวต์ระบุซื้อ และแอตทริบิวต์นี้พร้อมใช้งานเป็นบูลีน ดังนั้น เราไม่จำเป็นต้องเพิ่มคอลัมน์จากการคำนวณสำหรับป้ายชื่อ ในชุดข้อมูลอื่น ๆ คุณอาจต้องแปลงแอตทริบิวต์ป้ายชื่อที่มีอยู่ลงในคอลัมน์บูลีน

เลือกคำ**เสร็จสิ้น**ปุ่มเมื่อต้องปิดตัวแก้ไข Power Query ซึ่งแสดงรายการของเอนทิตีที่มีการ_ชม Online_เราได้เพิ่มข้อมูล เลือก**บันทึก**ที่มุมบนขวา ใส่ชื่อสำหรับ dataflow จากนั้น**บันทึก**บนกล่องโต้ตอบ ดังที่แสดงในรูปต่อไปนี้

![บันทึกการ dataflow](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-09.png)

### <a name="refresh-the-dataflow"></a>รีเฟรชกระแสข้อมูล

บันทึกผลลัพธ์ dataflow ในการแจ้งเตือนที่แสดง ระบุว่า dataflow ของคุณถูกบันทึก เลือก**รีเฟรชเดี๋ยวนี้**เพื่อรวบรวมข้อมูลจากแหล่งข้อมูลลงใน dataflow

![รีเฟรชเดี๋ยวนี้](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-10.png)

เลือก **ปิด** ที่มุมขวาบน และรอการรีเฟรชกระแสข้อมูลเพื่อเสร็จสิ้น

คุณยังสามารถรีเฟรช dataflow ของคุณ โดยใช้**การดำเนินการ**คำสั่งได้ Dataflow แสดงการประทับเวลาเมื่อรีเฟรชที่เสร็จสมบูรณ์

![ประทับเวลาของการรีเฟรช](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-11.png)

## <a name="create-and-train-a-machine-learning-model"></a>สร้าง และฝึกแบบจำลองการเรียนรู้เครื่อง

เลือก dataflow หลังจากการรีเฟรชเสร็จเรียบร้อย เมื่อต้องเพิ่มแบบเรียนรู้เครื่อง เลือก**ยที่นำไปใช้แบบจำลอง**ปุ่มในการ**การดำเนินการ**รายการสำหรับเอนทิตีพื้นฐานที่ประกอบด้วยการฝึกอบรมข้อมูลและป้ายชื่อข้อมูล จากนั้น**เพิ่มเป็น แบบจำลองการเรียนรู้เครื่อง**

![เพิ่มแบบจำลองการเรียนรู้ของเครื่อง](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-12.png)

ขั้นตอนแรกสำหรับการสร้างแบบจำลองการเรียนรู้ของเราเครื่องคือเพื่อ ระบุข้อมูลในอดีตรวมถึงเขตข้อมูลป้ายชื่อที่คุณต้องการทำนาย แบบจำลองจะถูกสร้างขึ้น ด้วยการเรียนรู้จากข้อมูลนี้

ในกรณีของชุดข้อมูลเรากำลังใช้ นี่คือการ**รายได้**เขตข้อมูล เลือก**รายได้**เป็นค่า 'เขตข้อมูลผลลัพธ์ในอดีต' แล้วเลือก**ถัดไป**

![เลือกข้อมูลในอดีต](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-13.png)

ถัดไป เราต้องเลือกชนิดของเครื่องเรียนรู้แบบจำลองเพื่อสร้าง Power BI วิเคราะห์ค่าในเขตข้อมูลผลลัพธ์ในอดีตที่คุณระบุ และแนะนำชนิดของแบบจำลองการเรียนรู้เครื่องที่สามารถสร้างการคาดการณ์เขตข้อมูลนั้น

ในกรณีนี้ เนื่องจากเรากำลังประมาณผลลัพธ์เป็นไบนารีของว่าผู้ใช้จะทำการสั่งซื้อ หรือไม่ เลือก**คาดการณ์ไบนารี**สำหรับชนิดแบบจำลอง และจากนั้นเลือกถัดไป

![คาดการณ์ไบนารีที่เลือก](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-14.png)

ถัดไป Power BI ไม่การสแกนเบื้องต้นของข้อมูล และป้อนข้อมูลที่สามารถใช้รูปแบบแนะนำ คุณสามารถเลือกที่จะกำหนดเขตข้อมูลป้อนเข้าที่ใช้สำหรับแบบจำลอง ในชุดข้อมูลของเรารวบรวม เมื่อต้องเลือกเขตข้อมูลทั้งหมด เลือกกล่องกาเครื่องหมายถัดจากชื่อเอนทิตี เลือก**ถัดไป**เพื่อยอมรับการป้อนข้อมูล

![เลือกกล่องกาเครื่องหมายถัดไป](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-15.png)

ในขั้นตอนสุดท้าย เราต้องใส่ชื่อสำหรับแบบจำลองของเรา และป้ายชื่อเรียกง่ายสำหรับผลลัพธ์ที่จะใช้ในรายงานสร้างขึ้นโดยอัตโนมัติซึ่งจะสรุปผลลัพธ์ของการตรวจสอบแบบจำลอง ถัดไป เราจำเป็นต้องตั้งชื่อแบบจำลอง_คาดเดาจุดประสงค์ซื้อ_และป้ายชื่อ true และ false เป็น_ซื้อ_และ_ไม่ซื้อ_ จากนั้นเลือก**บันทึก**

![บันทึกแบบจำลอง](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-16.png)

รูปแบบของเราเรียนรู้เครื่องในขณะนี้พร้อมสำหรับการฝึกอบรม เลือก**รีเฟรชเดี๋ยวนี้**เพื่อเริ่มการฝึกอบรมแบบจำลอง

![รีเฟรชเดี๋ยวนี้](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-17.png)

กระบวนการฝึกอบรมจะเริ่มต้น โดยการสุ่มตัวอย่าง และนอร์มัลไลซ์ข้อมูลของคุณในอดีต และแยกชุดข้อมูลของคุณลงในเอนทิตีใหม่สอง*ข้อมูลการฝึกอบรมคาดเดาจุดประสงค์ซื้อ*และ*ซื้อเจตนาคาดการณ์ทดสอบ ข้อมูล*

ขึ้นอยู่กับขนาดของชุดข้อมูล กระบวนการฝึกอบรมอาจใช้เวลาใดก็ได้จากสักครู่เพื่อสองสามชั่วโมง ในตอนนี้ คุณสามารถดูรูปแบบในการ**เครื่องเรียนรู้แบบจำลอง**แท็บของ dataflow ได้ การ_พร้อม_สถานะระบุว่า แบบจำลองมีอยู่ในคิวสำหรับการฝึกอบรม หรืออยู่ระหว่างการฝึกอบรม

![พร้อมสำหรับการฝึกอบรม](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-18.png)

ในขณะที่การฝึกอบรม คุณจะไม่สามารถดู หรือแก้ไข dataflow คุณสามารถยืนยันว่า แบบจำลองกำลังฝึก และตรวจสอบสถานะของ dataflow ที่ผ่านได้ ซึ่งปรากฏเป็นการรีเฟรชข้อมูลในความคืบหน้าในการ**Dataflows**แท็บของพื้นที่ทำงาน

![ในกระบวนการ](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-19.png)

เมื่อการฝึกอบรมแบบจำลองเสร็จสมบูรณ์ dataflow แสดงเวลาการรีเฟรชอัปเดตแล้ว คุณสามารถยืนยันได้ว่า รูปแบบที่ได้รับการฝึกฝน โดยไป**เรียนรู้แบบจำลองเครื่อง**แท็บใน dataflow ได้ คุณสร้างแบบจำลองควรแสดงสถานะเป็น**Trained**และ**สุดท้ายฝึกฝน**ควรถูกปรับปรุงเวลาในขณะนี้

![ได้รับการฝึกอบรมบนสุด](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-20.png)

## <a name="review-the-model-validation-report"></a>ตรวจสอบรายงานการตรวจสอบแบบจำลอง

เพื่อตรวจสอบรายงานการตรวจสอบแบบจำลอง ในการ**โมเดลเรียนรู้เครื่อง s**เลือก**ดูรายงานประสิทธิภาพการทำงาน และนำไปใช้แบบจำลอง**ปุ่ม**การดำเนินการ**คอลัมน์สำหรับแบบจำลอง . รายงานนี้อธิบายวิธีแบบจำลองการเรียนรู้เครื่องของคุณมีแนวโน้มที่จะดำเนินการ

ในการ**ประสิทธิภาพการทำงานแบบจำลอง**หน้าของรายงาน เลือก**Influencers คีย์**เพื่อดู predictors ด้านบนสำหรับแบบจำลองของคุณ คุณสามารถเลือกหนึ่ง predictors เพื่อดูวิธีการแจกแจงผลลัพธ์เชื่อมโยงกับว่านั้น

![ประสิทธิภาพการทำงานแบบจำลอง](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-21.png)

คุณสามารถใช้**ขีดจำกัดความน่าเป็น**ตัวแบ่งส่วนข้อมูลบนหน้าประสิทธิภาพการทำงานแบบจำลองการตรวจสอบอิทธิพลความแม่นยำและเรียกคืนสำหรับแบบจำลองได้

![ค่าเกณฑ์ของความน่าเป็น](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-22.png)

หน้าของรายงานอธิบายวัดประสิทธิภาพทางสถิติสำหรับแบบจำลอง

รายงานยังมีหน้ารายละเอียดการฝึกอบรมที่อธิบายเกิดซ้ำอื่นที่ถูกเรียกใช้ วิธีคุณลักษณะถูกแยกจากการป้อนข้อมูล และ hyperparameters สำหรับแบบจำลองขั้นสุดท้ายที่ใช้

## <a name="apply-the-model-to-a-dataflow-entity"></a>นำรูปแบบไปใช้กับเอนทิตี dataflow

เลือกคำ**นำไปใช้แบบจำลอง**ปุ่มที่ด้านบนของรายงานเพื่อเรียกใช้แบบจำลองนี้เมื่อมีการรีเฟรช dataflow ในการ**นำไปใช้**โต้ตอบ คุณสามารถระบุเอนทิตีเป้าหมายที่มีข้อมูลต้นฉบับซึ่งควรนำไปใช้แบบจำลอง ได้

![นำแบบจำลองไปใช้](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-23.png)

เมื่อได้รับพร้อมท์ คุณต้อง**รีเฟรช**dataflow เพื่อแสดงตัวอย่างผลลัพธ์ของแบบจำลองของคุณ

นำรูปแบบจะสร้างเอนทิตีใหม่ ด้วยคำต่อท้าย**เลี่ยง < model_name >** ผนวกเข้ากับเอนทิตีที่คุณนำไปใช้แบบจำลอง ในกรณีของเรา แบบจำลองที่จะนำไปใช้ใน**OnlineShoppers**จะสร้างเอนทิตี**OnlineShoppers เลี่ยงคาดเดาจุดประสงค์ซื้อ**ซึ่งรวมถึงผลลัพธ์ที่คาดไว้จากแบบจำลอง

นำไปใช้แบบจำลองการคาดการณ์ไบนารีเพิ่มสามคอลัมน์ที่ มีผลลัพธ์ที่คาดไว้ คะแนนความน่าเป็น และ influencers ระเบียนเฉพาะด้านบนสำหรับการคาดการณ์ แต่ละคำที่ขึ้นต้น ด้วยชื่อคอลัมน์ที่ระบุ

![คอลัมน์ที่สามของผลลัพธ์](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-24.png)

เนื่องจากปัญหาทราบแล้ว คอลัมน์ผลลัพธ์ได้คะแนนในเอนทิตีที่สมบูรณ์ได้เท่านั้นสามารถเข้าถึงได้จาก Power BI Desktop เมื่อต้องการแสดงตัวอย่างเหล่านี้ในบริการ คุณต้องใช้เอนทิตีอย่างพิเศษ

เมื่อรีเฟรช dataflow เสร็จสมบูรณ์ คุณสามารถเลือก**OnlineShoppers เลี่ยงแสดงตัวอย่างการคาดเดาจุดประสงค์ซื้อ**เอนทิตีเพื่อดูผลลัพธ์ได้

![ดูผลลัพธ์](media/service-tutorial-build-machine-learning-model/tutorial-build-machine-learning-model-25.png)

## <a name="using-the-scored-output-from-the-model-in-a-power-bi-report"></a>ใช้ผลลัพธ์ได้คะแนนจากแบบจำลองในรายงาน Power BI

เมื่อต้องใช้การแสดงผลได้คะแนนจากเครื่องของคุณเรียนรู้แบบจำลอง คุณสามารถเชื่อมต่อกับ dataflow ของคุณจาก Power BI desktop ใช้ตัวเชื่อมต่อ Dataflows การ**OnlineShoppers เลี่ยงคาดเดาจุดประสงค์ซื้อ**ตอนนี้คุณสามารถใช้เอนทิตีเพื่อรวมการคาดการณ์จากแบบจำลองของคุณในรายงาน Power BI ได้
