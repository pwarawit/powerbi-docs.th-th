# <a name="monitor-power-bi-premium-and-power-bi-embedded-capacities"></a>ตรวจสอบความจุของ Power BI Premium และ Power BI Embedded

บทความนี้แสดงภาพรวมของการตรวจสอบตัววัดความจุ Power BI Premium ของคุณ การตรวจสอบการใช้งานความจจะุช่วยให้คุณสามารถใช้วิธีจัดการความจุได้อย่างมีข้อมูล

คุณสามารถตรวจสอบความจุด้วยแอปการวัดความจุ Power BI Premium หรือในพอร์ทัลผู้ดูแลระบบ เราขอแนะนำให้ใช้แอป เนื่องจากจะมีรายละเอียดมาก แต่บทความนี้ครอบคลุมทั้งสองตัวเลือก

**เวอร์ชันปัจจุบันของแอปคือ 1.10 (เผยแพร่เมื่อ 13 ธันวาคม 2018)**

.

## <a name="install-the-premium-capacity-metrics-app"></a>ติดตั้ง แอปวัดความจุ Premium

คุณสามารถไปยัง[แอปวัดความจุ Premium ](https://app.powerbi.com/groups/me/getapps/services/capacitymetrics)ได้โดยตรงหรือติดตั้งแอปเหมือนที่คุณทำกับแอปอื่นๆ ใน Power BI

1. ใน Power BI คลิ๊ก**แอป**

    ![ไปที่แอป](media/service-admin-premium-monitor-capacity/apps.png)

1. ทางด้านขวา คลิ๊ก**รับแอป**

1. ในประเภท**แอป** ค้นหา**แอปวัดความจุ Power BI Premium**

1. สมัครใช้งานเพื่อติดตั้งแอป

หลังจากที่คุณติดตั้งแอป คุณสามารถดูตัววัดในเรื่องความจุในองค์กรของคุณ ลองมาดูการวัดสำคัญบางอย่างที่พร้อมใช้งาน

## <a name="use-the-metrics-app"></a>ใช้แอปการวัด

| **ข้อมูลสรุปของชุดข้อมูล** | * จำนวนรวมของชุดข้อมูลในพื้นที่ทำงานทั้งหมดในความจุของคุณ<br> * จำนวนครั้งที่มีการเชื่อมต่อคิวรีโดยตรง/การเชื่อมต่อสดเกิน 80% ของค่าเกณฑ์ในเจ็ดวันที่ผ่านมา แยกเป็นบักเก็ตสามนาที<br> * เวลาส่วนใหญ่ที่มีการเชื่อมต่อคิวรีโดยตรง/การเชื่อมต่อสดเกิน 80% ในช่วงเจ็ดวันที่ผ่านมา แยกเป็นบักเก็ตหนึ่งชั่วโมง<br> * เวลาท้องถิ่นที่มีการชื่อมต่อคิวรีโดยตรง/การเชื่อมต่อสดเกิน 80% มากที่สุดในหนึ่งชั่วโมง<br> * จำนวนทั้งหมดของการรีเฟรชใน 7 วันที่ผ่านมา<br> * เวลารอรีเฟรชเฉลี่ย - การหน่วงเวลาเฉลี่ยระหว่างเวลาที่กำหนดไว้และเวลาเริ่มต้นการรีเฟรช หน่วยเป็นนาที<br> * ระยะเวลาการรีเฟรชเฉลี่ย - เวลาที่ใช้ในการรีเฟรชจนเสร็จสิ้น หน่วยเป็นนาที<br> * จำนวนรวมของคิวรีทั้งหมดที่เรียกใช้ใน 7 วันที่ผ่านมา<br> * เวลารอคิวรีเฉลี่ย - เวลาที่คิวรีรอบนทรัพยากรของระบบก่อนเริ่มการเรียกใช้งาน หน่วยเป็นมิลลิวินาที<br> ระยะเวลาคิวรีโดยเฉลี่ย - เวลาที่ใช้ในการคิวรีจนเสร็จสิ้น หน่วยเป็นมิลลิวินาที<br> * จำนวนรวมของแบบจำลองที่ลดลงเนื่องจากหน่วยความจำไม่เพียงพอ<br> * ขนาดเฉลี่ยของชุดข้อมูล <br> * จำนวนเฉลี่ยของชุดข้อมูลที่โหลดลงในหน่วยความจำ |

### <a name="metrics-report"></a>รายงานเมทริกซ์

คลิกแดชบอร์ดเพื่อไปยังรายงานพื้นฐาน รายงานมีห้าแท็บ ดังจะอธิบายรายละเอียดเพิ่มเติมในส่วนด้านล่าง

* **ชุดข้อมูล**: เมตริกโดยละเอียดของสถานภาพของชุดข้อมูล Power BI ภายในความจุของคุณ

* **รายงานแบบแบ่งหน้า**: เมทริกซ์โดยละเอียดเกี่ยวกับสถานภาพของรายงานแบบแบ่งหน้าภายในความจุของคุณ

* **กระแสข้อมูล**: เมตริกโดยละเอียดของการรีเฟรชสำหรับกระแสข้อมูลในความจุของคุณ

* **การใช้ทรัพยากร**: เมทริกซ์ความจุโดยรวม รวมทั้งหน่วยความจำและการใช้งาน CPU แบบสูง

* **รหัสและข้อมูล**: ชื่อ รหัส และเจ้าของของความจุ พื้นที่ทำงาน และปริมาณงาน

ในแต่ละแท็บ คุณสามารถกรองเมทริกซ์ตามความจุและช่วงวันได้ หากไม่ได้เลือกตัวกรอง รายงานตามค่าเริ่มต้นจะแสดงผลเมทริกซ์ของสัปดาห์ที่ผ่านมาสำหรับความจุทั้งหมดที่เป็นเมทริกซ์การรายงาน
