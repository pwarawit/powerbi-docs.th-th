# <a name="use-report-themes-in-power-bi-desktop"></a>ใช้ธีมรายงานใน Power BI Desktop
ด้วย**ธีมรายงาน** คุณสามารถใช้ธีมสี กับรายงานของคุณทั้งหมด เช่น สีขององค์กร การกำหนดสีตามฤดูกาล หรือธีมสีอื่นๆ ที่คุณอาจต้องการนำไปใช้ เมื่อคุณนำ**ธีมรายงาน**ไปใช้งาน วิชวลทั้งหมดในรายงานของคุณจะใช้สีจากธีมที่คุณเลือก (มีข้อยกเว้นบางข้อ ซึ่งจะอธิบายภายหลังในบทความนี้)

![](media/desktop-report-themes/report-themes_1.png)

การนำ**ธีมรายงาน**ไปใช้ จำเป็นต้องใช้ไฟล์ JSON ซึ่งมีโครงสร้างพื้นฐาน ซึ่งคุณสามารถนำเข้าลงใน Power BI Desktop แล้วนำไปใช้กับรายงานของคุณ โครงสร้างไฟล์ JSON และกระบวนการในการนำเข้า (เพียงการคลิกไม่กี่ปุ่ม) ทำได้ง่าย และรวดเร็ว

เริ่มตั้งแต่การเผยแพร่ เดือนกันยายน 2017 ของ **Power BI Desktop** คุณสามารถกำหนดองค์ประกอบธีมรายงานเพิ่มมากขึ้นอีกโดยใช้ไฟล์ JSON และกำหนดด้วยตนเอง (และกำหนดมาตรฐาน) องค์ประกอบเกือบทั้งหมดด้วยไฟล์ JSON ที่คุณสามารถปรับด้วยตนเองในบานหน้าต่าง**การจัดรูปแบบ** เป้าหมายของการเผยแพร่ เดือนกันยายน 2017 (และภายหลัง) คือการเปิดให้คุณสามารถควบคุมการแสดงรายงานของคุณอย่างเต็มที่ ไปจนถึงระดับแยกย่อยได้

### <a name="enable-report-themes"></a>เปิดใช้งานธีมรายงาน
รุ่นเดือนสิงหาคม 2018 ของ**Power BI Desktop**การกำหนดธีมรายงานจะพร้อมใช้งานโดยทั่วไป และไม่จำเป็นต้องเปิดใช้งาน 

ถ้าคุณกำลังใช้**Power BI Desktop** เวอร์ชันก่อนหน้าคุณสามารถเปิดใช้งานเป็นคุณลักษณะตัวอย่างโดยเลือก**ไฟล์ > ตัวเลือกและการตั้งค่า > ตัวเลือก > คุณลักษณะตัวอย่าง** จากนั้นเลือกกล่องกาเครื่องหมายด้านข้าง**ธีมรายงานแบบกำหนดเอง** คุณจะต้องรีสตาร์ท **Power BI Desktop** หลังจากที่คุณทำการเลือกแล้ว

![](media/desktop-report-themes/report-themes_2.png)

เราแนะนำให้อัปเกรดเป็นเวอร์ชันล่าสุด**Power BI Desktop**เสมอ ซึ่งคุณสามารถรับได้จากลิงก์ใน[รับ Power BI Desktop](desktop-get-the-desktop.md) 

## <a name="how-report-themes-work"></a>ธีมรายงาน ทำงานอย่างไร
หากต้องการใช้ธีมรายงานกับรายงาน Power BI Desktop คุณสามารถเลือกจากธีมรายงานที่มีอยู่แล้วภายใน หรือนำเข้าธีมแบบกำหนดเอง

หากต้องเลือกจากธีมรายงานที่มีอยู่แล้วภายใน ให้เลือกปุ่ม**สลับธีม**จากริบบิ้น**หน้าแรก** จากนั้นเลือกธีมที่มีอยู่จากเมนูดรอปดาวน์

![เลือกธีมรายงาน](media/desktop-report-themes/report-themes_2a.png)

ธีมรายงานของคุณจะถูกนำไปใช้กับรายงาน และคุณก็พร้อมแล้ว

### <a name="importing-report-themes"></a>การนำเข้าธีมรายงาน

หากต้องการนำเข้าธีมรายงาน ให้เลือกปุ่ม**สลับธีม** จากริบบิ้น**หน้าแรก** จากนั้นเลือก**นำเข้าธีม**จากรายการแบบดรอปดาวน์

![](media/desktop-report-themes/report-themes_3.png)

หน้าต่างจะปรากฏขึ้น ให้คุณนำทางไปยังตำแหน่งที่ตั้งของไฟล์ธีม JSON Power BI Desktop ค้นหาไฟล์ JSON ซึ่งเป็นไฟล์ชนิดธีมรายงาน Power BI ในรูปต่อไปนี้ แสดงธีมวันหยุดจำนวนหนึ่งที่มี เราจะเลือกธีมวันหยุดหนึ่งในเดือนมีนาคม

![](media/desktop-report-themes/report-themes_4.png)

เมื่อโหลดไฟล์ธีมเสร็จเรียบร้อยแล้ว Power BI Desktop จะบอกให้คุณทราบ

![](media/desktop-report-themes/report-themes_5.png)

ตอนนี้เราได้นำเข้าไฟล์ธีมแล้ว ลองมาดูโครงสร้างไฟล์ JSON ที่เรียบง่าย และตรงไปตรงมา

## <a name="structure-of-a-report-theme-json-file"></a>โครงสร้างของไฟล์ธีมรายงาน JSON
ไฟล์ JSON พื้นฐานที่เลือกในส่วนที่แล้ว (ไฟล์ *St Patricks Day.json*) เมื่อเปิดในตัวแก้ไข จะมีลักษณะดังนี้

![](media/desktop-report-themes/report-themes_6.png)

ไฟล์ JSON ต้องมีบรรทัดต่อไปนี้:

* **name** - นี่คือชื่อธีม ซึ่งเป็นเขตข้อมูลเดียวที่จำเป็น
* **dataColors** - รายการรหัสสีรหัสฐานสิบหกที่ใช้เป็นข้อมูลในวิชวล Power BI Desktop รายการสามารถมีจำนวนสี มากหรือน้อยตามความต้องการ
* **background**, **foreground** และ **tableAccent** - ค่าเหล่านี้คือ สีที่ใช้ในวิชวลประเภทต่าง ๆ **foreground** ใช้กับข้อความในกล่องข้อความ, ข้อความเป้าหมาย KPI, ข้อความการ์ดหลายแถว, ข้อความค่าการ์ด, คำบรรยายมาตรวัด, องค์ประกอบข้อความของตัวแบ่งส่วนข้อมูลแนวตั้ง และ ﻿﻿ข้อความผลรวมและค่าใน**ตาราง**และ**เมทริกซ์** **background** ใช้เพื่อเติมสีปุ่ม และพื้นหลังป้ายชื่อแผนภูมิผสม วิธีที่ใช้สีเหล่านี้ขึ้นอยู่สไตล์วิชวลที่ถูกนำไปใช้ วิชวล**ตาราง**และ**เมทริกซ์** นำสไตล์เหล่านี้ไปใช้ตามค่าเริ่มต้น

เพื่อใช้สไตล์วิชวล**ตาราง**หรือ**เมทริกซ์** เลือกวิชวล และในบานหน้าต่าง**การจัดรูปแบบการแสดงข้อมูล** เลือกส่วน**รูปแบบ** แล้วขยาย**รูปแบบเมทริกซ์** และเลือกสไตล์จากรายการดรอปดาวน์**สไตล์**

![](media/desktop-report-themes/report-themes_7.png)

เพื่อให้ง่ายต่อการคัดลอกและวาง เพื่อสร้างไฟล์ JSON ของคุณเอง นี่คือ เนื้อหาของไฟล์ *St Patricks Day.json*:

    {
        "name": "St Patricks Day",
        "dataColors": ["#568410", "#3A6108", "#70A322", "#915203", "#D79A12", "#bb7711", "#114400", "#aacc66"],
        "background":"#FFFFFF",
        "foreground": "#3A6108",
        "tableAccent": "#568410"
    }

จากจุดนั้น คุณสามารถใส่รหัสฐานสิบหกของสีของคุณเองสำหรับสีที่คุณเลือก

เริ่มตั้งแต่การเผยแพร่ เดือนกันยายน 2017 ของ **Power BI Desktop** ไฟล์ JSON สามารถปรับได้ละเอียดขึ้น ในไฟล์ JSON คุณเพียงแค่กำหนดรูปแบบที่คุณต้องการเปลี่ยน และรูปแบบอื่น ๆ ที่*ไม่*ได้กำหนดในไฟล์ JSON ของคุณจะกลับไปใช้ค่าเริ่มต้นของ Power BI

ประโยชน์ของการสร้างไฟล์ JSON มีหลายข้อ ตัวอย่างเช่น คุณสามารถระบุว่า แผนภูมิทั้งหมดใช้ขนาดแบบอักษร 12 หรือว่า วิชวลบางวิชวลให้ใช้ตระกูลแบบอักษรที่กำหนด หรือปิดใช้งานป้ายชื่อสำหรับแผนภูมิบางชนิด

ด้วยความสามารถในการใช้ไฟล์ JSON ในระดับแยกย่อย คุณสามารถสร้างไฟล์ธีมที่กำหนดมาตรฐานให้แผนภูมิและรายงานของคุณ ทำให้รายงานขององค์กรมีรูปแบบเดียวกันได้ง่าย

สำหรับข้อมูลเกี่ยวกับรูปแบบของไฟล์ JSON โดยละเอียด ดู**รูปแบบไฟล์ธีมรายงาน JSON** ที่ส่วนท้ายของบทความนี้

## <a name="how-report-theme-colors-stick-to-your-reports"></a>สีจากธีมรายงานติดไปกับรายงานของคุณอย่างไร
เมื่อคุณเผยแพร่รายงานของคุณไปยัง **บริการของ Power BI** จะมีสีของธีมรายงานของคุณติดไปด้วย

นอกจากนี้ ส่วน**สีของข้อมูล** ของแผง**รูปแบบ** สะท้อนให้เห็นธีมรายงานของคุณ ตัวอย่างเช่น หลังจากใช้โทนสีเขียวและสีน้ำตาลจากธีม**วันเซนต์แพทริก** เมื่อเราเลือกวิชวลและไปที่**รูปแบบ > สีของข้อมูล** เราจะเห็นต่อไปนี้

![](media/desktop-report-themes/report-themes_8.png)

เห็นสีเขียวทั้งหมดนั่นไหม? เพราะว่าสีเหล่านั้นเป็นส่วนหนึ่งของ**ธีมรายงาน** ที่เรานำเข้าและใช้งาน

### <a name="situations-when-report-theme-colors-wont-stick-to-your-reports"></a>สถานการณ์เมื่อสีจากธีมรายงานไม่ติดไปกับรายงานของคุณ
ถ้าคุณใช้ ชุดสี (หรือสี) แบบกำหนดเอง กับบางจุดข้อมูลในวิชวล เมื่อนำธีมรายงานมาใช้ จะ*ไม่*แทนที่สีของจุดข้อมูลแบบกำหนดเองนั้น

นอกจากนี้ ถ้าคุณได้ตั้งค่าสีของจุดข้อมูลเอง โดยใช้ส่วนสีธีม จากชุดแบบสี สีนั้น (หรือหลายสีนั้น) จะ*ไม่*ปรับปรุงเมื่อคุณนำธีมรายงานใหม่มาใช้ เพื่อให้คุณได้ใช้ค่าเริ่มต้นของสีตามเดิม (เพื่อจะได้ปรับปรุงเมื่อคุณใช้ธีมรายงานใหม่) คุณสามารถเลือก**กลับไปใช้ค่าเริ่มต้น**ในชุดแบบสี**สีของธีม**

![](media/desktop-report-themes/report-themes_9.png)

นอกจากนี้ **วิชวลแบบกำหนดเอง**หลายวิชวลจะไม่ใช้ธีมรายงาน

## <a name="report-theme-files-you-can-use-right-now"></a>ไฟล์ธีมรายงานที่คุณสามารถใช้ในขณะนี้
ต้องการเริ่มต้นใช้**ธีมรายงาน**หรือไม่? เยี่ยม! ต่อไปนี้คือไฟล์ JSON ธีมรายงานสำเร็จบางไฟล์ที่คุณสามารถดาวน์โหลด และนำเข้าไปยังรายงาน **Power BI Desktop** ของคุณ พร้อมกับรูปภาพของธีมรายงานที่นำไปใช้กับรายงานที่ใช้ในบทความนี้

* [ธีม](https://go.microsoft.com/fwlink/?linkid=843924)ที่ใช้ใน[บล็อกโพสต์](https://powerbi.microsoft.com/blog/power-bi-desktop-march-feature-summary/)ที่ประกาศการเผยแพร่ครั้งแรกของ**ธีมรายงาน** ที่เรียกว่า [*waveform.json*](https://go.microsoft.com/fwlink/?linkid=843924)

![](media/desktop-report-themes/report-themes_10.png)

* [ธีมที่เหมาะกับผู้มีความบกพร่องทางการมองเห็น](https://go.microsoft.com/fwlink/?linkid=843923)มากกว่าธีมสีเริ่มต้น เรียกว่า[*ColorblindSafe-Longer.json*](https://go.microsoft.com/fwlink/?linkid=843923)

![](media/desktop-report-themes/report-themes_11.png)

* [ธีม Power View](https://go.microsoft.com/fwlink/?linkid=843925) จำนวนมากในไฟล์ zip รวมถึงธีมที่ชื่อว่า [*Apothecary.json*](https://go.microsoft.com/fwlink/?linkid=843925) ที่แสดงด้านล่างนี้

![](media/desktop-report-themes/report-themes_12.png)

* สุดท้าย นี่คือธีมที่ดูน่ารัก (ธีม*วันวาเลนไทน์*) และแน่นอนว่าจะดึงดูดความสนใจของคุณ

![](media/desktop-report-themes/report-themes_13.png)

แทนที่จะเป็นการดาวน์โหลด นี่คือรหัสสำหรับไฟล์ JSON วันวาเลนไทน์:

    {
        "name": "Valentine's Day",
        "dataColors": ["#990011", "#cc1144", "#ee7799", "#eebbcc", "#cc4477", "#cc5555", "#882222", "#A30E33"],
        "background":"#FFFFFF",
        "foreground": "#ee7799",
        "tableAccent": "#990011"
    }

**ธีมรายงาน**สามารถทำให้รายงาน Power BI Desktop ของคุณ สะท้อนสีสันในแบบของคุณ องค์กรของคุณ หรือแม้แต่ฤดูกาล หรือวันหยุดได้ สนุกกับมัน เป็นเรื่องง่ายมาก​!

## <a name="report-theme-json-file-format"></a>รูปแบบไฟล์ธีมรายงาน JSON
ไฟล์ JSON พื้นฐาน มีห้าบรรทัดที่จำเป็น:

* **name** - นี่คือชื่อธีม ซึ่งเป็นเขตข้อมูลเดียวที่จำเป็น
* **dataColors** - รายการรหัสสีรหัสฐานสิบหกที่ใช้เป็นข้อมูลในวิชวล Power BI Desktop รายการสามารถมีจำนวนสี มากหรือน้อยตามความต้องการ
* **background**, **foreground** และ **tableAccent** - ค่าเหล่านี้คือ สีที่ใช้ในวิชวลประเภทต่าง ๆ **foreground** ใช้กับข้อความในกล่องข้อความ, ข้อความเป้าหมาย KPI, ข้อความการ์ดหลายแถว, ข้อความค่าการ์ด, คำบรรยายมาตรวัด, องค์ประกอบข้อความของตัวแบ่งส่วนข้อมูลแนวตั้ง และ ﻿﻿ข้อความผลรวมและค่าใน**ตาราง**และ**เมทริกซ์** **background** ใช้เพื่อเติมสีปุ่ม และพื้นหลังป้ายชื่อแผนภูมิผสม วิธีที่ใช้สีเหล่านี้ขึ้นอยู่สไตล์วิชวลที่ถูกนำไปใช้ วิชวล**ตาราง**และ**เมทริกซ์** นำสไตล์เหล่านี้ไปใช้ตามค่าเริ่มต้น

เพื่อสร้างไฟล์ JSON รูปแบบขยาย ที่มีการควบคุมรูปแบบที่ละเอียดขึ้น และในระดับแยกย่อย คุณจำเป็นต้องเพิ่มส่วน **visualStyles** ให้กับไฟล์ JSON และกำหนดรูปแบบซ้อนอยู่ในส่วนนั้น รูปแบบของส่วน **visualStyles** มีลักษณะดังต่อไปนี้:

    visualStyles: {
        visualName: {
            styleName: {
                cardName: [{
                    propertyName: propertyValue
                }]
            }
        }
    }

สำหรับส่วน **visualName** และ **cardName** คุณสามารถกำหนดรายการของวิชวลหรือชื่อการ์ด หรือคุณสามารถใช้เครื่องหมายดอกจัน ("\*") ถ้าคุณต้องการให้การตั้งค่าการนำไปใช้กับวิชวลทั้งหมด หรือการ์ดทั้งหมดของวิชวลที่มีคุณสมบัติที่ระบุ คุณสามารถใช้เครื่องหมายดอกจัน ("\*") เมื่อคุณต้องการตั้งค่าส่วนกลางในรายงานของคุณ เช่น ขนาดแบบอักษร หรือตระกูลแบบอักษร ที่กำหนด ให้กับทุกวิชวลในรายงานของคุณได้

> [!NOTE]
> คุณระบุแค่องค์ประกอบการจัดรูปแบบที่คุณต้องการแก้ไขเท่านั้น องค์ประกอบการจัดรูปแบบใด ๆ ที่ไม่ได้ระบุในไฟล์ JSON จะกลับไปใช้ค่าและการตั้งค่าเริ่มต้น
> 
>
