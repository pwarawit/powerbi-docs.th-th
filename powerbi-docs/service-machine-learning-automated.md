# <a name="automated-machine-learning-in-power-bi-preview"></a>การเรียนรู้ของเครื่องอัตโนมัติใน Power BI (ดูตัวอย่าง)

การเรียนรู้ด้วยเครื่องอัตโนมัติ (AutoML) สำหรับกระแสข้อมูลช่วยนักวิเคราะห์ทางธุรกิจในการฝึกตรวจสอบและเรียกใช้แบบจำลอง Machine Learning โดยตรงใน Power BI ซึ่งรวมถึงประสบการณ์ง่ายๆ ในการสร้างรูปแบบ ML ใหม่ซึ่งนักวิเคราะห์สามารถใช้กระแสข้อมูลของพวกเขาเพื่อระบุข้อมูลป้อนเข้าสำหรับการฝึกแบบจำลอง บริการจะแยกคุณลักษณะที่เกี่ยวข้องมากที่สุดโดยอัตโนมัติเลือกอัลกอริทึมและเพลงที่เหมาะสมและตรวจสอบรูปแบบ ML หลังจากที่แบบจำลองได้รับการฝึกแล้ว Power BI จะสร้างรายงานที่มีผลลัพธ์ของการตรวจสอบความถูกต้องที่อธิบายประสิทธิภาพและผลลัพธ์ให้กับนักวิเคราะห์ จากนั้นแบบจำลองจะถูกเรียกใช้ในข้อมูลใหม่หรืออัปเดตข้อมูลภายใน กระแสข้อมูล

![หน้าจอการเรียนรู้ของเครื่อง](media/service-machine-learning-automated/automated-machine-learning-power-bi-01.png)

การเรียนรู้เครื่องอัตโนมัติพร้อมใช้งานสำหรับกระแสข้อมูลที่โฮสต์บน Power BI Premium และความจุแบบฝังตัวเท่านั้น ในการแสดงตัวอย่างนี้ AutoML ช่วยให้คุณสามารถฝึกแบบจำลองการเรียนรู้ของเครื่องสำหรับการคาดการณ์ไบนารีการจัดประเภทและการถดถอย

## <a name="working-with-automl"></a>การทำงานกับ AutoML

[กระแสข้อมูล Power BI](service-dataflows-overview.md)นำเสนอการเตรียมข้อมูลด้วยตนเองสำหรับข้อมูลขนาดใหญ่ AutoML ช่วยให้คุณสามารถใช้ประโยชน์จากความพยายามในการเตรียมข้อมูลของคุณสำหรับการสร้างรูปแบบการเรียนรู้ของเครื่องได้โดยตรงภายใน Power BI

AutoML ใน Power BI ช่วยนักวิเคราะห์ข้อมูลเพื่อใช้กระแสข้อมูลในการสร้างโมเดลการเรียนรู้ของเครื่องด้วยประสบการณ์การใช้งานแบบง่ายด้วยทักษะ Power BI เท่านั้น ส่วนใหญ่ของวิทยาศาสตร์ข้อมูลเบื้องหลังการสร้างรูปแบบ ML เป็นอัตโนมัติโดย Power BI โดยใช้ guardrails เพื่อให้แน่ใจว่ารูปแบบที่ผลิตมีคุณภาพที่ดีและการมองเห็นเพื่อให้คุณมีข้อมูลเชิงลึกเต็มรูปแบบในกระบวนการที่ใช้ในการสร้างรูปแบบ ML ของคุณ

AutoML สนับสนุนการสร้าง**การคาดการณ์ไบนารี**แบบจำลอง**การจัดประเภท**และ**การถดถอย**สำหรับกระแสข้อมูล เหล่านี้คือประเภทของแบบจำลองการเรียนรู้เครื่องที่ได้รับการดูแลซึ่งหมายความว่าพวกเขาได้เรียนรู้จากผลลัพธ์ที่ทราบแล้วของการสังเกตการณ์ที่ผ่านมาเพื่อคาดการณ์ผลลัพธ์ของการสังเกตการณ์อื่นๆ ชุดข้อมูลป้อนเข้าสำหรับการฝึกแบบจำลอง AutoML คือชุดของระเบียนที่มี**ป้ายชื่อ**ที่มีผลลัพธ์ที่ทราบ

AutoML ใน Power BI รวม [MLอัตโนมัติ](https://docs.microsoft.com/azure/machine-learning/service/concept-automated-ml)จาก[Azure Machine Learning service](https://docs.microsoft.com/azure/machine-learning/service/overview-what-is-azure-ml)เพื่อสร้างรูปแบบ ML ของคุณ อย่างไรก็ตามคุณไม่จำเป็นต้องมีการสมัครใช้งาน Azure เพื่อให้ AutoML ใน Power BI กระบวนการของการฝึกและโฮสต์รูปแบบ ML ได้รับการจัดการทั้งหมดโดยบริการของ Power BI

หลังจากแบบจำลอง ML ได้รับการฝึก AutoML ให้สร้างรายงาน Power BI โดยอัตโนมัติที่อธิบายประสิทธิภาพการทำงานของรูปแบบ ML ของคุณ AutoML เน้นความสามารถในการอธิบาย โดยการเน้นที่ปัจจัยที่มีอิทธิพลในการป้อนข้อมูลของคุณที่มีผลต่อการคาดการณ์ที่ส่งกลับโดยแบบจำลองของคุณ รายงานยังรวมถึงเมตริกหลักสำหรับแบบจำลองโดยขึ้นอยู่กับชนิดแบบจำลอง ML

หน้าอื่นๆ ของรายงานที่สร้างขึ้นแสดงสรุปทางสถิติของแบบจำลองและรายละเอียดการฝึกอบรม สรุปทางสถิติคือความสนใจของผู้ใช้ที่ต้องการดูหน่วยวัดประสิทธิภาพของข้อมูลมาตรฐานสำหรับแบบจำลอง รายละเอียดการฝึกสรุปการเกิดซ้ำทั้งหมดที่มีการเรียกใช้เพื่อสร้างแบบจำลองของคุณด้วยพารามิเตอร์การสร้างโมเดลที่เกี่ยวข้อง นอกจากนี้ยังอธิบายวิธีการป้อนข้อมูลแต่ละครั้งที่ใช้ในการสร้างแบบจำลอง ML

จากนั้นคุณสามารถนำรูปแบบ ML ของคุณไปใช้กับข้อมูลของคุณสำหรับการให้คะแนนได้ เมื่อกระแสข้อมูลได้รับการรีเฟรชการคาดการณ์จากรูปแบบ ML ของคุณจะถูกนำไปใช้กับข้อมูลของคุณโดยอัตโนมัติ Power BI ยังรวมถึงคำอธิบายเฉพาะสำหรับคะแนนการคาดการณ์แต่ละรายการที่มีรูปแบบ ML สร้างขึ้น

## <a name="creating-a-machine-learning-model"></a>การสร้างแบบจำลองการเรียนรู้ของเครื่อง

ในส่วนนี้จะอธิบายวิธีการสร้างแบบจำลองการเรียนรู้ AutoML 

### <a name="data-prep-for-creating-an-ml-model"></a>การเตรียมข้อมูลสำหรับการสร้างรูปแบบ ML

ในการสร้างแบบจำลองการเรียนรู้ของเครื่องใน Power BI คุณต้องสร้าง กระแสข้อมูลสำหรับข้อมูลที่มีข้อมูลผลลัพธ์ในอดีตซึ่งใช้สำหรับการฝึกแบบจำลอง ML สำหรับรายละเอียดเกี่ยวกับการกำหนดค่ากระแสข้อมูล ของคุณให้ดูที่[การเตรียมข้อมูลแบบบริการตนเองใน Power BI](service-dataflows-overview.md)

ในการเผยแพร่ปัจจุบัน Power BI จะใช้ข้อมูลจากเฉพาะเอนทิตีเดียวเพื่อฝึกแบบจำลอง ML ดังนั้นถ้าข้อมูลในอดีตของคุณประกอบด้วยหลายเอนทิตีคุณต้องเข้าร่วมข้อมูลลงในเอนทิตีกระแสข้อมูลเดียวด้วยตนเอง นอกจากนี้คุณควรเพิ่มคอลัมน์จากการคำนวณสำหรับเมตริกธุรกิจใดๆ ที่อาจมีทำนายที่แข็งแกร่งสำหรับผลลัพธ์ที่คุณกำลังพยายามคาดการณ์

AutoML มีข้อกำหนดข้อมูลเฉพาะสำหรับการฝึกแบบจำลองการเรียนรู้ของเครื่อง ข้อกำหนดเหล่านี้จะอธิบายไว้ในส่วนด้านล่างตามชนิดแบบจำลองที่เกี่ยวข้อง

### <a name="configuring-the-ml-model-inputs"></a>การกำหนดค่าอินพุทแบบจำลอง ML

หากต้องการสร้างแบบจำลอง AutoML ให้เลือกไอคอน MLในคอลัมน์ **การดำเนินกา**รของเอนทิตีกระแสข้อมูล ด้วยข้อมูลในอดีตและเลือก**เพิ่มแบบจำลองการเรียนรู้ของเครื่อง**

![เพิ่มแบบจำลองการเรียนรู้ของเครื่อง](media/service-machine-learning-automated/automated-machine-learning-power-bi-02.png)

มีการเปิดใช้งานประสบการณ์ที่ง่ายดายซึ่งประกอบด้วยตัวช่วยสร้างที่แนะนำคุณในกระบวนการสร้างรูปแบบ ML ตัวช่วยสร้างมีขั้นตอนง่ายๆดังต่อไปนี้

1. เลือกเอนทิตีที่มีข้อมูลผลลัพธ์ในอดีตและเขตข้อมูลที่คุณต้องการการคาดการณ์
2. เลือกชนิดแบบจำลองตามชนิดของการคาดการณ์ที่คุณต้องการดู
3. เลือกข้อมูลป้อนเข้าที่คุณต้องการให้รูปแบบใช้เป็นสัญญาณที่คาดเดาได้
4. ตั้งชื่อแบบจำลองของคุณและบันทึกการกำหนดค่าของคุณ

เขตข้อมูลผลลัพธ์ในอดีตจะระบุแอตทริบิวต์ป้ายชื่อสำหรับการฝึกแบบจำลอง ML ที่แสดงในรูปต่อไปนี้

![เลือกข้อมูลผลลัพธ์ในอดีต](media/service-machine-learning-automated/automated-machine-learning-power-bi-03.png)

เมื่อคุณระบุผลลัพธ์เขตข้อมูลที่ได้ในอดีต AutoML วิเคราะห์ข้อมูลป้ายชื่อเพื่อระบุชนิดของรูปแบบ ML ที่สามารถรับการฝึกฝนสำหรับข้อมูลดังกล่าวและแนะนำชนิดแบบจำลอง ML ที่เป็นไปได้ที่สามารถรับการฝึกได้ 

> [!NOTE]
> แบบจำลองบางชนิดอาจไม่ได้รับการสนับสนุนสำหรับข้อมูลที่คุณเลือก

นอกจากนี้ AutoML ยังวิเคราะห์เขตข้อมูลทั้งหมดในเอนทิตีที่เลือกเพื่อแนะนำอินพุทที่สามารถใช้สำหรับการฝึกแบบจำลอง ML ได้ กระบวนการนี้จะเป็นค่าประมาณและยึดตามการวิเคราะห์ทางสถิติดังนั้นคุณควรตรวจทานข้อมูลที่ใช้ในการป้อนข้อมูล อินพุตใดๆที่ขึ้นอยู่กับเขตข้อมูลผลลัพธ์ในอดีต (หรือเขตข้อมูลป้ายชื่อ) ไม่ควรใช้สำหรับการฝึกแบบจำลอง ML เนื่องจากจะส่งผลกระทบต่อประสิทธิภาพการทำงาน

![กำหนดค่าเขตข้อมูลอินพุต](media/service-machine-learning-automated/automated-machine-learning-power-bi-04.png)

ในขั้นตอนสุดท้ายคุณสามารถตั้งชื่อแบบจำลองและบันทึกค่า

ในขั้นตอนนี้คุณจะได้รับพร้อมท์ให้รีเฟรชกระแสข้อมูล ซึ่งเริ่มกระบวนการฝึกหัดสำหรับแบบจำลอง ML

### <a name="ml-model-training"></a>การฝึกแบบจำลอง ML

การฝึกแบบจำลอง AutoML เป็นส่วนหนึ่งของรีเฟรชกระแสข้อมูล ก่อนอื่น AutoML เตรียมข้อมูลของคุณสำหรับการฝึก

AutoML จะแยกข้อมูลในอดีตที่คุณให้ไว้ในชุดรูปแบบการฝึกอบรมและการทดสอบ ชุดข้อมูลทดสอบคือชุด holdout ที่ใช้สำหรับการตรวจสอบประสิทธิภาพของแบบจำลองหลังจากการฝึก สิ่งเหล่านี้จะถูกรับรู้ว่ามี**การฝึกและการทดสอบ**เอนทิตีในกระแสข้อมูล AutoML ใช้การตรวจสอบความถูกต้องข้ามสำหรับการตรวจสอบแบบจำลอง

ถัดไปแต่ละเขตข้อมูลป้อนเข้าจะได้รับการวิเคราะห์และใช้การใส่ค่า ซึ่งแทนที่ค่าที่ขาดหายไปด้วยค่าที่มีการแทนที่ มีการใช้กลยุทธ์การใส่ค่าที่แตกต่างกันสองตัวโดย AutoML จากนั้นจะมีการใช้การสุ่มตัวอย่างใดๆ ที่จำเป็นและการปรับสภาพข้อมูลของคุณ

AutoML ใช้การแปลงหลายรายการแต่ละเขตข้อมูลป้อนเข้าที่เลือกโดยยึดตามชนิดข้อมูลและคุณสมบัติทางสถิติ AutoML ใช้การแปลงข้อมูลเหล่านี้เพื่อแยกคุณลักษณะสำหรับใช้ในการฝึกแบบจำลอง ML ของคุณ

กระบวนการฝึกสำหรับแบบจำลอง AutoML ประกอบด้วยการทำซ้ำ 50 ครั้งที่มีการวนกลับกับอัลกอริทึมการสร้างโมเดลที่แตกต่างกันและการตั้งค่า hyperparameter เพื่อค้นหาแบบจำลองที่มีประสิทธิภาพที่ดีที่สุด ประสิทธิภาพของแต่ละโมเดลเหล่านี้จะถูกประเมินโดยการตรวจสอบความถูกต้องของชุดข้อมูลทดสอบ holdout ในระหว่างขั้นตอนการฝึกอบรมนี้ AutoML สร้างไปป์ไลน์หลายตัวสำหรับการฝึกอบรมและการตรวจสอบการเกิดซ้ำเหล่านี้ กระบวนการประเมินประสิทธิภาพการทำงานของแบบจำลองสามารถใช้เวลาได้จากหลายนาทีถึงสองสามชั่วโมงโดยขึ้นอยู่กับขนาดของชุดข้อมูลของคุณและทรัพยากรความจุเฉพาะที่พร้อมใช้งาน

ในบางกรณีแบบจำลองขั้นสุดท้ายที่สร้างขึ้นอาจใช้การเรียนรู้ในวงการที่มีการใช้หลายแบบเพื่อให้ประสิทธิภาพการทำงานที่ดีขึ้น

### <a name="automl-model-explainability"></a>ความสามารถในการอธิบายแบบจำลอง AutoML

หลังจากที่ได้รับการฝึกแบบจำลอง AutoML วิเคราะห์ความสัมพันธ์ระหว่างคุณลักษณะการป้อนข้อมูลและการแสดงผลแบบจำลอง AutoML ประเมินขนาดและทิศทางของการเปลี่ยนแปลงไปยังเอาต์พุตแบบจำลองสำหรับชุดข้อมูลทดสอบ holdout สำหรับแต่ละคุณลักษณะการป้อนข้อมูล ซึ่งเรียกว่า*ความสำคัญของคุณลักษณะ*

![ความสำคัญของคุณลักษณะ](media/service-machine-learning-automated/automated-machine-learning-power-bi-05.png)

### <a name="automl-model-report"></a>รายงานแบบจำลอง AutoML

AutoML สร้างรายงาน Power BI ที่สรุปประสิทธิภาพการทำงานของแบบจำลองระหว่างการตรวจสอบความถูกต้องพร้อมกับความสำคัญของคุณลักษณะทั่วโลก รายงานสรุปผลลัพธ์จากการใช้แบบจำลอง ML ไปยังข้อมูลทดสอบ holdout และเปรียบเทียบการคาดการณ์ด้วยค่าผลลัพธ์

คุณสามารถตรวจสอบรายงานแบบจำลองเพื่อทำความเข้าใจประสิทธิภาพ นอกจากนี้คุณยังสามารถตรวจสอบว่าปัจจัยหลักที่มีอิทธิพลของแบบจำลองอยู่ใกล้กับกับข้อมูลเชิงลึกทางธุรกิจเกี่ยวกับผลลัพธ์ที่รู้ผลอยู่แล้ว

แผนภูมิและหน่วยวัดที่ใช้เพื่ออธิบายประสิทธิภาพของแบบจำลองในรายงานจะขึ้นอยู่กับชนิดแบบจำลอง แผนภูมิประสิทธิภาพและหน่วยวัดเหล่านี้จะอธิบายไว้ในส่วนต่อไปนี้

หน้าเพิ่มเติมในรายงานอาจอธิบายหน่วยวัดทางสถิติเกี่ยวกับแบบจำลองจากมุมมองวิทยาศาสตร์ข้อมูล ตัวอย่างเช่นรายงาน**การคาดการณ์ไบนารี**จะมีแผนภูมิกำไรและเส้นโค้ง ROC สำหรับแบบจำลอง

รายงานยังรวมถึงหน้า**รายละเอียดการฝึก**ที่มีคำอธิบายว่าแบบจำลองได้รับการฝึกอบรมอย่างไรและมีแผนภูมิที่อธิบายประสิทธิภาพของแบบจำลองในแต่ละการทำซ้ำการดำเนินการ

![รายละเอียดการฝึกแบบจำลอง](media/service-machine-learning-automated/automated-machine-learning-power-bi-06.png)

อีกส่วนหนึ่งในหน้านี้จะอธิบายวิธีการใส่ค่า วิธีการที่ใช้สำหรับการกรอกค่าที่ขาดหายไปสำหรับเขตข้อมูลป้อนเข้าเช่นเดียวกับวิธีการป้อนข้อมูลแต่ละเขตเพื่อแยกคุณลักษณะที่ใช้ในแบบจำลอง นอกจากนี้ยังมีพารามิเตอร์ที่ใช้โดยแบบจำลองขั้นสุดท้าย

![รายละเอียดเพิ่มเติมสำหรับแบบจำลอง](media/service-machine-learning-automated/automated-machine-learning-power-bi-07.png)

ถ้าแบบจำลองที่สร้างขึ้นใช้การเรียนรู้ทั้งหมด ในหน้า**รายละเอียดการฝึก**ยังรวมถึงส่วนที่อธิบายน้ำหนักของแต่ละรูปแบบของส่วนประกอบในวงดนตรีเช่นเดียวกับพารามิเตอร์

![น้ำหนักทั้งหมด](media/service-machine-learning-automated/automated-machine-learning-power-bi-08.png)

## <a name="applying-the-automl-model"></a>การใช้แบบจำลอง AutoML

ถ้าคุณพอใจกับประสิทธิภาพการทำงานของรูปแบบ ML ที่สร้างขึ้นคุณสามารถนำไปใช้กับข้อมูลใหม่หรืออัปเดตเมื่อมีการรีเฟรช กระแสข้อมูลของคุณ คุณสามารถดำเนินการได้จากรายงานแบบจำลองโดยการเลือกปุ่ม**นำไปใช้**ที่มุมขวาบน

เมื่อต้องการใช้รูปแบบจำลอง ML คุณต้องระบุชื่อของเอนทิตีที่จะต้องใช้และคำนำหน้าสำหรับคอลัมน์ที่จะถูกเพิ่มไปยังเอนทิตีนี้สำหรับการแสดงผลแบบจำลอง คำนำหน้าเริ่มต้นสำหรับชื่อคอลัมน์คือชื่อแบบจำลอง ฟังก์ชันการ*นำไปใช้*อาจรวมพารามิเตอร์เพิ่มเติมเฉพาะสำหรับชนิดแบบจำลอง

การใช้รูปแบบ ML จะสร้างเอนทิตี dataflow ใหม่ด้วยคำต่อท้าย**ที่เต็มไปด้วย < model_name >** ตัวอย่างเช่นถ้าคุณใช้โมเดล_PurchaseIntent_ไปยังเอนทิตี _OnlineShoppers_ ผลลัพธ์จะสร้าง **PurchaseIntentที่เต็มไปด้วย OnlineShoppers**

ในปัจจุบันเอนทิตีเอาท์พุทไม่สามารถใช้เพื่อแสดงตัวอย่างผลลัพธ์แบบจำลอง ML ในตัวแก้ไข Power Query คอลัมน์ผลลัพธ์จะแสดงค่า null เป็นผลลัพธ์เสมอ เมื่อต้องการดูผลลัพธ์เอนทิตีเอาท์พุทที่สองที่มีส่วนต่อท้ายที่**เต็มไปด้วย < model_name > การแสดงตัวอย่าง**ที่มีการสร้างขึ้นเมื่อมีการใช้แบบจำลอง

คุณจะต้องรีเฟรชกระแสข้อมูล เพื่อแสดงตัวอย่างผลลัพธ์ Query Editor ของคุณ

![ตัวแก้ไขคิวรี](media/service-machine-learning-automated/automated-machine-learning-power-bi-09.png)

เมื่อคุณใช้แบบจำลอง AutoML จะเก็บการคาดการณ์ของคุณให้เป็นปัจจุบันเสมอเมื่อมีการรีเฟรชกระแสข้อมูล

AutoML ยังรวมถึงคำอธิบายเฉพาะสำหรับแต่ละแถวที่มีคะแนนในเอนทิตีเอาต์พุต

หากต้องการใช้ข้อมูลเชิงลึกและการคาดการณ์จากรูปแบบ ML ในรายงาน Power BI คุณสามารถเชื่อมต่อไปยังเอนทิตีเอาต์พุตจาก Power BI Desktop โดยใช้ตัวเชื่อมต่อ**กระแสข้อมูล**

## <a name="binary-prediction-models"></a>แบบจำลองการทำนายแบบไบนารี

แบบจำลองการคาดการณ์ไบนารีที่รู้จักกันอย่างเป็นทางการเป็น**แบบจำลองการจัดประเภทไบนารี**จะใช้เพื่อจัดประเภทชุดข้อมูลลงในสองกลุ่ม แบบจำลองการจัดประเภทไบนารีใช้เพื่อคาดการณ์เหตุการณ์ที่สามารถมีผลลัพธ์แบบไบนารีเช่นว่าโอกาสทางการขายจะแปลงหรือไม่ไม่ว่าจะเป็นบัญชีผู้ใช้จะได้รับการชำระเงินในเวลาหรือไม่ ไม่ว่าจะเป็นการฉ้อฉลธุรกรรมและอื่นๆ

เนื่องจากผลลัพธ์คือไบนารี Power BI คาดว่าป้ายชื่อสำหรับแบบจำลองการคาดการณ์ไบนารีเป็นบูลีนซึ่งมีผลที่ได้รับการติดป้ายชื่อ **true**หรือ **false** ตัวอย่างเช่นในรูปแบบการแปลงโอกาสทางการขายโอกาสทางการขายที่ได้รับการชนะจะมีป้ายชื่อเป็น true ผู้ที่สูญเสียจะมีป้ายชื่อเท็จและโอกาสการขายที่เปิดอยู่มีป้ายชื่อเป็น null

ผลลัพธ์ของแบบจำลองการคาดการณ์ไบนารีคือคะแนนความน่าจะเป็นซึ่งระบุความเป็นไปได้ว่าผลลัพธ์ที่สอดคล้องกับค่าป้ายชื่อเป็นจริงจะสำเร็จ

### <a name="training-a-binary-prediction-model"></a>แบบจำลองการทำนายแบบไบนารี

เมื่อต้องการสร้างแบบจำลองการคาดการณ์ไบนารีเอนทิตีการป้อนข้อมูลที่มีการฝึกอบรมของคุณต้องมีเขตข้อมูลบูลีนเป็นเขตข้อมูลผลลัพธ์ในอดีตเพื่อระบุผลลัพธ์ที่ทราบดีอยู่แล้ว

การเตรียมปัจจัยที่จำเป็น:

* เขตข้อมูลบูลีนต้องใช้เป็นเขตข้อมูลผลลัพธ์ในอดีต
* ต้องมีอย่างน้อย 50 แถวของข้อมูลในอดีตสำหรับแต่ละระดับชั้นของผลลัพธ์

โดยทั่วไปถ้าผลลัพธ์ที่ผ่านมาจะถูกระบุโดยเขตข้อมูลของชนิดข้อมูลที่แตกต่างกันคุณสามารถเพิ่มคอลัมน์จากการคำนวณเพื่อแปลงสิ่งเหล่านี้เป็นบูลีนโดยใช้ Power Query

กระบวนการสร้างสำหรับแบบจำลองการคาดการณ์ไบนารีจะเป็นไปตามขั้นตอนเดียวกันกับแบบจำลองAutoML อื่นๆที่อธิบายไว้ในส่วนที่**กำหนดค่าอินพุทแบบจำลอง ML** ด้านบน

### <a name="binary-prediction-model-report"></a>รายงานแบบจำลองการทำนายแบบไบนารี

แบบจำลองการคาดการณ์ไบนารีสร้างเป็นการแสดงผลเป็นความน่าจะเป็นที่ระเบียนจะบรรลุผลที่กำหนดโดยค่าป้ายชื่อบูลีนเป็น True รายงานมีตัวแบ่งส่วนข้อมูลสำหรับขีดจำกัดความน่าจะเป็นซึ่งมีผลต่อวิธีการที่คะแนนด้านบนและด้านล่างของขีดจำกัดความน่าจะเป็น

รายงานอธิบายประสิทธิภาพการทำงานของแบบจำลองในแง่ของ *True Positives*, *False Positives*, *True Negatives* และ*False Negatives*. True Positive และ True Negative ได้รับการคาดการณ์ผลอย่างถูกต้องสำหรับสองชั้นในข้อมูลผลลัพธ์ False Positive คือผลลัพธ์ที่มีป้ายชื่อบูลีนจริงของค่าเท็จแต่ได้รับการคาดการณ์ว่าเป็น True ในทางกลับกันFalse Negative จะเป็นผลมาจากที่ค่าป้ายชื่อบูลีนที่แท้จริงเป็นจริงแต่ได้รับการคาดการณ์เป็นเท็จ

หน่วยวัดเช่นความแม่นยำและการเรียกใช้ให้อธิบายผลกระทบของขีดจำกัดน่าจะเป็นที่มาของผลลัพธ์ที่คาดการณ์ไว้ คุณสามารถใช้ตัวแบ่งส่วนข้อมูลของขีดจำกัดความน่าจะเป็นเพื่อเลือกขีดจำกัดที่ทำให้มีความสมดุลระหว่างความแม่นยำและการเรียกคืนได้

![การแสดงตัวอย่างความแม่นยำ](media/service-machine-learning-automated/automated-machine-learning-power-bi-10.png)

หน้า**รายงานที่แม่นยำ**ของรายงานแบบจำลองมีแผนภูมิ*กำไรสะสม*และเส้นโค้ง ROC สำหรับแบบจำลอง รายงานเหล่านี้คือหน่วยวัดทางสถิติของประสิทธิภาพแบบจำลอง รายงานประกอบด้วยคำอธิบายของแผนภูมิที่แสดง

![หน้าจอรายงานความแม่นยำ](media/service-machine-learning-automated/automated-machine-learning-power-bi-11.png)

### <a name="applying-a-binary-prediction-model"></a>การนำแบบจำลองการทำนายแบบไบนารีไปใช้

เมื่อต้องการใช้แบบจำลองการคาดการณ์ไบนารีคุณต้องระบุเอนทิตีที่มีข้อมูลที่คุณต้องการใช้การคาดการณ์จากรูปแบบ ML พารามิเตอร์อื่นๆรวมถึงคำนำหน้าชื่อคอลัมน์ผลลัพธ์และขีดจำกัดความน่าจะเป็นสำหรับการจัดประเภทผลลัพธ์ที่คาดการณ์ไว้

![อินพุทการคาดการณ์](media/service-machine-learning-automated/automated-machine-learning-power-bi-12.png)

เมื่อมีการใช้แบบจำลองการคาดการณ์ไบนารีจะมีการเพิ่มคอลัมน์ขาออกที่สามไปยังเอนทิตีที่มีผลลัพธ์สมบูรณ์กว่า ข้อมูลเหล่านี้คือ**PredictionScore**, **PredictionOutcome**และ**PredictionExplanation** ชื่อคอลัมน์ในเอนทิตีมีการระบุคำนำหน้าเมื่อใช้แบบจำลอง

คอลัมน์**PredictionOutcome**ประกอบด้วยป้ายชื่อผลลัพธ์ที่คาดการณ์ไว้ ระเบียนที่มีความน่าจะเป็นเกินขีดจำกัดได้รับการคาดการณ์เป็นแนวโน้มที่จะทำให้เกิดผลลัพธ์และผู้ที่อยู่ด้านล่างได้รับการคาดการณ์ว่าไม่น่าจะบรรลุผล

คอลัมน์ **PredictionExplanationประกอบ**ด้วยคำอธิบายที่มีผลกระทบเฉพาะที่คุณลักษณะป้อนเข้าที่มีใน **PredictionScore** นี่คือคอลเลกชันที่จัดรูปแบบ JSON ของตุ้มน้ำหนักของคุณลักษณะการป้อนข้อมูลสำหรับการคาดการณ์

## <a name="classification-models"></a>แบบจำลองการจัดประเภท

แบบจำลองการจัดประเภทจะใช้ในการแยกประเภทชุดข้อมูลลงในกลุ่มหรือประเภทในหลายรายการ  แบบจำลองการจัดประเภทใช้เพื่อคาดการณ์เหตุการณ์ที่สามารถมีหนึ่งในผลลัพธ์ที่เป็นไปได้หลายอย่างเช่นว่าลูกค้ามีแนวโน้มที่จะมีค่าสูงสุดสูงปานกลางหรือต่ำ ความเสี่ยงสำหรับค่าเริ่มต้นคือสูง, ปานกลาง, ต่ำหรือต่ำสุดหรือไม่ และอื่นๆ

ผลลัพธ์ของแบบจำลองการจัดประเภทคือคะแนนความน่าจะเป็นซึ่งระบุถึงความเป็นไปได้ว่าระเบียนจะบรรลุเกณฑ์สำหรับคลาสที่กำหนด

### <a name="training-a-classification-model"></a>การฝึกแบบจำลองการจัดประเภท

การป้อนข้อมูลที่มีการฝึกอบรมของคุณสำหรับแบบจำลองการจัดประเภทต้องมีเขตข้อมูลตัวเลขเป็นเขตข้อมูลผลลัพธ์ในอดีตเพื่อระบุผลลัพธ์ที่ทราบดีอยู่แล้ว

การเตรียมปัจจัยที่จำเป็น:

* ต้องมีอย่างน้อย 50 แถวของข้อมูลในอดีตสำหรับแต่ละระดับชั้นของผลลัพธ์

กระบวนการสร้างสำหรับแบบจำลองการคาดการณ์จะเป็นไปตามขั้นตอนเดียวกันกับแบบจำลองAutoML อื่นๆที่อธิบายไว้ในส่วนที่**กำหนดค่าอินพุทแบบจำลอง ML** ด้านบน

### <a name="classification-model-report"></a>รายงานผลลัพธ์แบบจำลองการจัดประเภท

รายงานแบบจำลองการจัดประเภทได้รับการผลิตโดยใช้รูปแบบ ML เป็นข้อมูลทดสอบ holdout และเปรียบเทียบคลาสที่คาดการณ์สำหรับระเบียนที่มีคลาสที่รู้จักจริง

รายงานแบบจำลองมีแผนภูมิที่มีการแบ่งย่อยของระเบียนที่มีการจัดประเภทอย่างถูกต้องและไม่ถูกต้องสำหรับแต่ละคลาสที่รู้จัก

![รายงานแบบจำลอง](media/service-machine-learning-automated/automated-machine-learning-power-bi-13.png)

การเลื่อนเฉพาะระดับชั้นเพิ่มเติมจะช่วยให้การวิเคราะห์วิธีการกระจายการคาดการณ์สำหรับคลาสที่รู้จัก ซึ่งรวมถึงคลาสที่อื่นๆที่บันทึกของคลาสที่รู้จักมีแนวโน้มที่จะจัดประเภทผิด

![รายงานการวิเคราะห์](media/service-machine-learning-automated/automated-machine-learning-power-bi-14.png)

คำอธิบายแบบจำลองในรายงานซึ่งรวมไปถึงการคาดการณ์ระดับสูงสำหรับแต่ละคลาส

รายงานแบบจำลองการจัดประเภทยังรวมถึงหน้ารายละเอียดการฝึกอบรมที่คล้ายกับหน้าสำหรับชนิดแบบจำลองอื่นๆตามที่อธิบายไว้ในส่วน**รายงานแบบจำลอง AutoML**ก่อนหน้าในบทความนี้

### <a name="applying-a-classification-model"></a>การนำแบบจำลองการจัดประเภทไปใช้

เมื่อต้องการใช้แบบจำลองการจัดประเภท ML คุณต้องระบุเอนทิตีที่มีข้อมูลป้อนเข้าและคำนำหน้าชื่อคอลัมน์ผลลัพธ์

เมื่อมีการใช้แบบจำลองการจัดประเภทจะมีการเพิ่มคอลัมน์ขาออกที่สามไปยังเอนทิตีที่มีผลลัพธ์สมบูรณ์กว่า ข้อมูลเหล่านี้คือ**PredictionScore**, **PredictionClass**และ**PredictionExplanation** ชื่อคอลัมน์ในเอนทิตีมีการระบุคำนำหน้าเมื่อใช้แบบจำลอง

คอลัมน์**PredictionClass**ประกอบด้วยคลาสที่คาดการณ์ไว้มากที่สุดสำหรับระเบียน คอลัมน์**PredictionScore**ประกอบด้วยรายการคะแนนความน่าจะเป็นสำหรับระเบียนสำหรับแต่ละคลาส

คอลัมน์ **PredictionExplanationประกอบ**ด้วยคำอธิบายที่มีผลกระทบเฉพาะที่คุณลักษณะป้อนเข้าที่มีใน **PredictionScore** นี่คือคอลเลกชันที่จัดรูปแบบ JSON ของตุ้มน้ำหนักของคุณลักษณะการป้อนข้อมูลสำหรับการคาดการณ์

## <a name="regression-models"></a>แบบจำลองการถดถอย

แบบจำลองการถดถอยจะใช้เพื่อทำนายค่าเช่นรายได้ที่น่าจะรับรู้จากข้อตกลงการขายค่าตลอดอายุการใช้งานของบัญชียอดเงินของใบแจ้งหนี้ลูกหนี้ที่มีแนวโน้มว่าจะชำระเงินวันที่ที่ใบแจ้งหนี้อาจได้รับชำระเงิน และอื่นๆ

ผลลัพธ์ของแบบจำลองการถดถอยคือค่าที่คาดการณ์ไว้

### <a name="training-a-regression-model"></a>การฝึกแบบจำลองการถดถอย

การป้อนข้อมูลที่มีการฝึกอบรมของคุณสำหรับแบบจำลองการถดถอยต้องมีเขตข้อมูลตัวเลขเป็นเขตข้อมูลผลลัพธ์ในอดีตเพื่อระบุค่าผลลัพธ์ที่ทราบดีอยู่แล้ว

การเตรียมปัจจัยที่จำเป็น:

* ต้องมีอย่างน้อย100 แถวของข้อมูลในอดีตสำหรับแบบจำลองการถดถอย

กระบวนการสร้างสำหรับแบบจำลองการการถดถอยจะเป็นไปตามขั้นตอนเดียวกันกับแบบจำลองAutoML อื่นๆที่อธิบายไว้ในส่วนที่**กำหนดค่าอินพุทแบบจำลอง ML** ด้านบน

### <a name="regression-model-report"></a>รายงานแบบจำลองการถดถอย

เช่นเดียวกับรายงานแบบจำลอง AutoML อื่นๆรายงานการถดถอยจะขึ้นอยู่กับผลลัพธ์จากการนำแบบจำลองไปใช้กับข้อมูลทดสอบ holdout

รายงานแบบจำลองมีแผนภูมิที่เปรียบเทียบค่าที่คาดการณ์ไว้กับค่าจริง ในแผนภูมินี้ระยะห่างจากเส้นทแยงมุมแสดงข้อผิดพลาดในการคาดการณ์

แผนภูมิข้อผิดพลาดที่เหลือแสดงการกระจายของเปอร์เซ็นต์ของข้อผิดพลาดเฉลี่ยสำหรับค่าที่แตกต่างกันในชุดข้อมูลการทดสอบ holdout แกนแนวนอนแสดงถึงค่าที่แท้จริงของกลุ่มที่มีขนาดของฟองที่แสดงความถี่หรือจำนวนของค่าในช่วงนั้น แกนแนวตั้งคือข้อผิดพลาดที่เหลือโดยเฉลี่ย

![แผนภูมิข้อผิดพลาดที่เหลืออยู่](media/service-machine-learning-automated/automated-machine-learning-power-bi-15.png)

รายงานแบบจำลองการถดถอยยังรวมถึงหน้ารายละเอียดการฝึกรายงานสำหรับชนิดแบบจำลองอื่นๆตามที่อธิบายไว้ในส่วน**รายงานแบบจำลอง AutoML**ด้านบน

### <a name="applying-a-regression-model"></a>การใช้แบบจำลองการถดถอย

เมื่อต้องการใช้แบบจำลองการถดถอย ML คุณต้องระบุเอนทิตีที่มีข้อมูลป้อนเข้าและคำนำหน้าชื่อคอลัมน์ผลลัพธ์

![นำการถดถอยมาใช้](media/service-machine-learning-automated/automated-machine-learning-power-bi-16.png)

เมื่อมีการใช้แบบจำลองการถดถอยจะมีการเพิ่มคอลัมน์ขาออกที่สองไปยังเอนทิตีที่มีผลลัพธ์สมบูรณ์กว่า ค่าเหล่านี้คือ**PredictionValue**และ**PredictionExplanation** ชื่อคอลัมน์ในเอนทิตีมีการระบุคำนำหน้าเมื่อใช้แบบจำลอง

คอลัมน์**PredictionValue**ประกอบด้วยค่าที่คาดการณ์ไว้สำหรับระเบียนที่ยึดตามเขตข้อมูลป้อนเข้า คอลัมน์ **PredictionExplanationประกอบ**ด้วยคำอธิบายที่มีผลกระทบเฉพาะที่คุณลักษณะป้อนเข้าค่าที่มีใน **PredictionScore** นี่คือคอลเลกชันที่จัดรูปแบบ JSON ของตุ้มน้ำหนักของคุณลักษณะการป้อนข้อมูล
