# <a name="key-influencers-visualization"></a>การแสดงภาพข้อมูลผู้มีอิทธิพลหลัก
Influencers คีย์ที่ภาพช่วยให้คุณเข้าใจปัจจัยไดรฟ์ที่คุณสนใจเมตริก ซึ่งช่วยวิเคราะห์ข้อมูลของคุณ จัดอันดับปัจจัยที่มีความสำคัญ และแสดงพวกเขาในฐานะที่เป็นผู้มีอิทธิพลหลัก ตัวอย่าง สมมติว่า คุณต้องการค้นหาว่ามีผลต่อสิ่งที่เรียกว่า churn หมุนเวียนพนักงาน ซึ่งก็คือ ปัจจัยหนึ่งอาจมีความยาวสัญญาการจ้างงาน และปัจจัยอื่นอาจมีอายุของพนักงาน 
 
## <a name="when-to-use-key-influencers"></a>เมื่อต้องใช้คีย์ influencers 
ภาพ influencers คีย์เป็นตัวเลือกที่ดีถ้าคุณต้องการ: 
- ดูปัจจัยที่ส่งผลต่อการวัดที่กำลังวิเคราะห์
- ความคมชัดความสำคัญของปัจจัยเหล่านี้ ตัวอย่างเช่น สัญญาระยะสั้นมีผลต่อการลาออกของพนักงานมากกว่าสัญญาระยะยาวหรือไม่? 

## <a name="key-influencer-requirements"></a>ข้อกำหนดของผู้มีอิทธิพลหลัก 
การวัดคุณวิเคราะห์ต้องเป็นตัวอักษร หรือตัวเลข (รวมและหน่วยวัดไม่สนับสนุน)

## <a name="features-of-the-key-influencers-visual"></a>คุณลักษณะของ influencers คีย์ที่ภาพ

![คุณลักษณะที่มีตัวเลข](media/power-bi-visualization-influencers/power-bi-ki-numbers-new.png)

1. **แท็บ**: เลือกแท็บเพื่อสลับระหว่างมุมมอง **แป้น influencers**แสดงผู้สนับสนุนสูงสุดเป็นค่าวัดที่เลือก **อันดับเซ็กเมนต์**แสดงเซกเมนต์ด้านบนที่มีส่วนร่วมกับค่าวัดที่เลือก *กลุ่ม* สร้างขึ้นจากชุดของค่า ตัวอย่าง เซ็กเมนต์หนึ่งอาจเป็นลูกค้าที่มีลูกค้าน้อย 20 ปี และอยู่ในภูมิภาคตะวันตก 

2. **กล่องดรอปดาวน์**: ค่าของการวัดภายใต้การตรวจสอบ ในตัวอย่างนี้ ค้นหาการวัด**จัดอันดับ** ค่าที่เลือกเป็น**ต่ำ**

3. **ย้อนหลัง**: ซึ่งช่วยให้คุณแปลภาพในบานหน้าต่างด้านซ้าย

4. **บานหน้าต่างด้านซ้าย**: บานหน้าต่างด้านซ้ายประกอบด้วยหนึ่งภาพ ในกรณีนี้ บานหน้าต่างด้านซ้ายแสดงรายการของ influencers คีย์ที่ด้านบน

5. **ย้อนหลัง**: ซึ่งช่วยให้คุณแปลภาพในบานหน้าต่างด้านขวา

6. **บานหน้าต่างด้านขวา**: บานหน้าต่างด้านขวาประกอบด้วยหนึ่งภาพ ในกรณีนี้ แผนภูมิคอลัมน์แสดงค่าทั้งหมดสำหรับ influencer คีย์**ธีม**ที่ถูกเลือกในบานหน้าต่างด้านซ้าย ค่าเฉพาะของ**ใช้งาน**จากบานหน้าต่างด้านซ้ายจะแสดงเป็นสีเขียว ค่าทั้งหมดอื่น ๆ สำหรับ**ธีม**จะแสดงเป็นสีดำ

7. **เส้นเฉลี่ย**: มีคำนวณค่าเฉลี่ยสำหรับค่าทั้งหมดอื่น ๆ เป็นไปได้สำหรับ**ธีม**ยกเว้น**ใช้งาน** ดังนั้นการคำนวณจึงใช้กับค่าทั้งหมดเป็นสีดำ ซึ่งบอกคุณว่าเปอร์เซ็นต์ของอีก**ธีม**มอบการจัดอันดับต่ำให้คุณ กล่าว เมื่อจัดอันดับถูกกำหนด โดยลูกค้า ลูกค้าที่ยังอธิบายสาเหตุหรือชุดรูปแบบของการจัดอันดับด้วย ธีมบางอย่างจะใช้งาน ความเร็ว และการรักษาความปลอดภัย 

   **ธีมคือการ ใช้งาน**คือ influencer คีย์วินาทีสูงสุดสำหรับการจัดอันดับต่ำ ตามรูปภาพในบานหน้าต่างด้านซ้าย ถ้าคุณหาค่าเฉลี่ยรูปแบบอื่น ๆ และมีส่วนร่วมของพวกเขาเพื่อการจัดอันดับ**ต่ำ**คุณได้รับผลลัพธ์ที่แสดงเป็นสีแดง ทั้งหมดอื่น ๆ ธีมที่กำหนด 11.35% จะสูงกว่า**ใช้งาน**

8. **กล่องกาเครื่องหมาย**: **แสดงเฉพาะ ค่าที่ influencers**

2. ย้ายการวัดที่คุณต้องการตรวจสอบลงในการ**วิเคราะห์**เขตข้อมูล การ**วิเคราะห์**เขตข้อมูลสนับสนุนประเภท หรือแบบไม่ต่อ เนื่อง ตัวแปรเท่านั้น เมื่อต้องดูลูกค้าไดรฟ์ใดเลือกการจัดอันดับของบริการให้น้อย**ตารางลูกค้า** > **จัดอันดับ** 
3. ย้ายเขตข้อมูลที่คุณคิดว่า อาจมีผลกระทบต่อ**จัดอันดับ**ลงในการ**อธิบายโดย**เขตข้อมูล คุณสามารถย้ายเขตข้อมูลได้มากเท่าที่คุณต้องการ ในกรณีนี้ เริ่มต้นด้วย:
    - ประเทศ-ภูมิภาค 
    - บทบาทในองค์กร 
    - ชนิดการสมัครใช้งาน 
    - ขนาดบริษัท 
    - ธีม 
1. เมื่อต้องเน้นการจัดอันดับลบ เลือก**ต่ำ**ในการ**สิ่งมีผลต่อการจัดอันดับต้อง**กล่องแบบหล่นลง  

    ![เลือกต่ำจากกล่องรายการดรอปดาวน์](media/power-bi-visualization-influencers/power-bi-key-influencers.png)

การวิเคราะห์การทำงานในระดับตารางของเขตข้อมูลที่กำลังวิเคราะห์ ในกรณีนี้ มีการ**จัดอันดับ**วัด เมตริกนี้ถูกกำหนดในระดับที่ลูกค้า ลูกค้าแต่ละรายได้รับคะแนนสูงสุดหรือคะแนนต่ำ ต้องกำหนดปัจจัยอธิบายทั้งหมดในระดับลูกค้าสำหรับวิชวลให้ใช้ของพวกเขา 

ในตัวอย่างก่อนหน้า ปัจจัยอธิบายทั้งหมดได้ไม่ว่าจะเป็นแบบหนึ่งต่อหนึ่งหรือหลายต่อหนึ่งความสัมพันธ์กับการวัด ในกรณีนี้ คะแนนแต่ละมีธีมหนึ่งที่เชื่อมโยงด้วย ธีมนี้มีธีมหลักของการตรวจทานลูกค้า ในทำนองเดียวกัน ลูกค้ามาจากประเทศหนึ่ง มีชนิดเป็นสมาชิกหนึ่ง และดำเนินการหนึ่งบทบาทในองค์กรของพวกเขา ปัจจัยอธิบายอยู่แอตทริบิวต์ของลูกค้า และไม่มีการแปลงจำเป็น ภาพสามารถทำให้ใช้ได้ทันที 

ในภายหลังในบทช่วยสอน คุณค้นหาที่ซับซ้อนมากขึ้นตัวอย่างที่มีความสัมพันธ์แบบหนึ่ง-ต่อกลุ่ม ในกรณีเหล่านั้น คอลัมน์ต้องการ รวมระดับลูกค้าก่อนที่คุณสามารถเรียกใช้การวิเคราะห์ 

หน่วยวัดและใช้เป็นปัจจัยอธิบายจะยังถูกประเมินในระดับตารางของการรวม**วิเคราะห์**วัด ตัวอย่างจะแสดงในภายหลังในบทความนี้ 

## <a name="interpret-categorical-key-influencers"></a>แปล influencers คีย์แบบกลุ่ม 
ลองมาดู influencers คีย์สำหรับการจัดอันดับต่ำ 

### <a name="top-single-factor-that-influences-the-likelihood-of-a-low-rating"></a>ปัจจัยเดียวที่มีผลต่อความน่าเป็นของการจัดอันดับต่ำสุด

องค์กรในตัวอย่างนี้มีบทบาทที่สาม: ผู้บริโภค ผู้ดูแล และผู้เผยแพร่ ถูกผู้บริโภคคือ ปัจจัยด้านบนที่ส่งผลกระทบต่อการจัดอันดับต่ำ 

![เลือกบทบาทในองค์กรเป็นผู้บริโภค](media/power-bi-visualization-influencers/power-bi-role-consumer.png)


เจาะจงขึ้นได้ ผู้ใช้ของคุณครั้ง 2.57 ดูเหมือนว่าจะให้คะแนนลบบริการของคุณ รายการแผนภูมิ influencers คีย์**บทบาทในองค์กรเป็นผู้บริโภค**แรกในรายการทางด้านซ้าย โดยการเลือก**บทบาทในองค์กรเป็นผู้บริโภค**, Power BI แสดงรายละเอียดเพิ่มเติมในบานหน้าต่างด้านขวา แสดงการเปรียบเทียบผลของแต่ละบทบาทของการจัดอันดับต่ำ
  
- 14.93% ของผู้บริโภคให้คะแนนต่ำ 
- โดยเฉลี่ย บทบาทอื่น ๆ ทั้งหมดให้คะแนนต่ำ 5.78% ของเวลา
- ผู้บริโภคครั้ง 2.57 ดูเหมือนว่าจะให้คะแนนต่ำเมื่อเทียบกับบทบาทอื่น ๆ ทั้งหมด คุณสามารถตรวจสอบนี้ โดยหารแถบสีเขียวด้วยเส้นประสีแดง 

### <a name="second-single-factor-that-influences-the-likelihood-of-a-low-rating"></a>ปัจจัยเดียวสองที่มีผลต่อความน่าเป็นของการจัดอันดับต่ำ

Influencers คีย์ที่ภาพเปรียบเทียบ และจัดลำดับปัจจัยจากตัวแปรต่าง ๆ มากมาย Influencer สองมีสิ่งใดทำงานร่วมกับ**บทบาทในองค์กร** เลือก influencer สองในรายการ ซึ่งก็คือ**ธีมคือการ ใช้งาน** 

![เลือกธีมคือการ ใช้งาน](media/power-bi-visualization-influencers/power-bi-theme.png)

ปัจจัยที่สองสำคัญเกี่ยวข้องกับชุดรูปแบบของการตรวจทานของลูกค้า ลูกค้าที่แสดงข้อคิดเห็นเกี่ยวกับการใช้งานของผลิตภัณฑ์ถูกครั้ง 2.55 อาจให้คะแนนต่ำเมื่อเทียบกับลูกค้าที่แสดงข้อคิดเห็นบนชุด เช่นความน่าเชื่อถือ ออกแบบ หรือความเร็ว 

ระหว่างวิชวล ค่าเฉลี่ย ซึ่งแสดง ด้วยเส้นประสีแดง เปลี่ยนจาก 5.78% 11.34% ค่าเฉลี่ยเป็นแบบไดนามิกเนื่องจากยึดตามค่าเฉลี่ยของค่าอื่น ๆ ทั้งหมด สำหรับ influencer แรก ค่าเฉลี่ยยกเว้นบทบาทลูกค้า สำหรับ influencer สอง ยังแยกธีมใช้งาน 
 
เลือกคำ**แสดงเฉพาะ ค่าที่ influencers**กล่องกาเครื่องหมายเพื่อกรองข้อมูล โดยใช้เฉพาะค่ามี ในกรณีนี้ พวกเขากำลังบทบาทที่ไดรฟ์คะแนนต่ำ ธีมสิบสองจะลดลงเป็นสี่ที่ Power BI ที่ถูกระบุว่าเป็นธีมที่กระตุ้นการจัดอันดับต่ำ 

![เลือกกล่องกาเครื่องหมาย](media/power-bi-visualization-influencers/power-bi-only-show.png)

## <a name="interact-with-other-visuals"></a>โต้ตอบกับวิชวลอื่น ๆ 
 
ทุกครั้งที่คุณเลือกตัวแบ่งส่วนข้อมูล ตัวกรอง หรือภาพอื่น ๆ บนพื้นที่ influencers คีย์ที่ภาพเรียกใช้การวิเคราะห์บนส่วนใหม่ของข้อมูล ตัวอย่าง คุณสามารถย้าย**ขนาดบริษัท**ลงในรายงาน และใช้เป็นตัวแบ่งส่วนข้อมูลได้ ใช้เพื่อดูถ้า influencers หลักสำหรับลูกค้าองค์กรของคุณจะแตกต่างจากจำนวนประชากรทั่วไป ขนาดบริษัทองค์กรมีขนาดใหญ่กว่าพนักงาน 50,000

## <a name="interpret-continuous-key-influencers"></a>แปล influencers คีย์อย่างต่อเนื่อง 
 
ดังนั้นสุด คุณเคยเห็นวิธีการใช้วิชวลเพื่อสำรวจเขตข้อมูลประเภทอื่นวิธีกำหนดการจัดอันดับต่ำ คุณจะยังสามารถมีปัจจัยต่อเนื่องเช่นอายุ ความสูง และราคาในการ**อธิบายโดย**เขตข้อมูล มาดูสิ่งที่เกิดขึ้นเมื่อ**อายุงาน**จะถูกย้ายจากตารางลูกค้าลงใน**อธิบายโดย** อายุงานมีลูกค้าระยะได้ใช้บริการ 
 
เมื่อเพิ่มอายุงาน ความน่าเป็นการรับการจัดอันดับด้านล่างเพิ่มขึ้น แนวโน้มนี้แนะนำให้ ลูกค้าดทำมีแนวโน้มมากเมื่อต้องให้คะแนนเป็นค่าลบ ข้อมูลเชิงลึกนี้จะน่าสนใจ และที่คุณอาจต้องการติดตามในภายหลัง 
 
แสดงภาพแสดงว่า ทุกครั้งที่อายุงานก็กลับเพิ่มขึ้นตามเดือน 13.44 โดยเฉลี่ยของการจัดอันดับต่ำเพิ่มเวลา 1.23 ในกรณีนี้ 13.44 เดือนแสดงถึงค่าเบี่ยงเบนมาตรฐานของระยะเวลาการครอบครอง เพื่อให้คุณได้รับข้อมูลเชิงลึกดูที่วิธีเพิ่มอายุงานมาตรฐานระยะ ซึ่งก็คือค่าเบี่ยงเบนมาตรฐานของอายุงาน มีผลต่อความเป็นไปได้รับการจัดอันดับต่ำ 
 
ลงจุดแผนภูมิกระจายในบานหน้าต่างด้านขวาลงจุดเปอร์เซ็นต์ค่าเฉลี่ยของการจัดอันดับต่ำสำหรับแต่ละค่าของอายุงาน ซึ่งเป็นเน้นความชัน ด้วยเส้นแนวโน้ม

## <a name="interpret-measures-and-aggregates-as-key-influencers"></a>แปลหน่วยวัดและรวมเป็น influencers คีย์ 
 
คุณสามารถใช้หน่วยวัดและรวมเป็นปัจจัยอธิบายภายในการวิเคราะห์ของคุณ ตัวอย่าง คุณอาจต้องการดูผลกระทบจำนวนตั๋วการสนับสนุนลูกค้าหรือระยะเวลาเฉลี่ยของตั๋วเปิดมีคะแนนคุณได้รับ 
 
ในกรณีนี้ คุณต้องการดูถ้าจำนวนใบอนุญาตที่มีลูกค้าที่มีผลต่อการให้พวกเขาให้คะแนน ในตอนนี้คุณนำเข้า**ID ตั๋วสนับสนุน**จากตารางตั๋วการสนับสนุน เนื่องจากลูกค้าสามารถมีใบอนุญาตหลาย คุณรวม ID ระดับลูกค้า รวมเป็นสิ่งสำคัญเนื่องจากการวิเคราะห์การทำงานระดับลูกค้า จึงต้องกำหนดโปรแกรมควบคุมทั้งหมดที่ระดับนั้นของส่วนประกอบ 
 
มาดูที่จำนวนของ Id ลูกค้าแต่ละแถวมีจำนวนตั๋วการสนับสนุนที่เกี่ยวข้องกับ ในกรณีนี้ เป็นจำนวนที่เพิ่มตั๋วสนับสนุน ความน่าเป็นของการจัดอันดับต่ำจะขึ้นครั้ง 5.51 ภาพทางด้านขวาแสดงค่าเฉลี่ยของจำนวนใบอนุญาต โดยอื่น**จัดอันดับ**ค่าที่ประเมินในระดับที่ลูกค้า

เซกเมนต์ด้านบนแสดงภาพรวมของเซ็กเมนต์ทั้งหมดที่ค้นพบ Power BI เริ่มต้น ตัวอย่างต่อไปนี้แสดงว่า พบเซ็กเมนต์หก เซ็กเมนต์เหล่านี้จะถูกจัดอันดับตามเปอร์เซ็นต์ของการจัดอันดับต่ำภายในเซกเมนต์ เซกเมนต์ 1 ตัวอย่าง มี 74.3% จัดอันดับลูกค้าที่มีสาย ยิ่งฟองสบู่สูงขึ้น สัดส่วนการจัดอันดับต่ำก็จะสูงขึ้นตาม ขนาดของฟองแสดงจำนวนลูกค้าที่อยู่ภายในเซกเมนต์

## <a name="working-with-numerical-data"></a>ทำงานกับข้อมูลตัวเลข

ถ้าคุณย้ายเขตข้อมูลตัวเลขไปยัง**วิเคราะห์**ฟิลด์ คุณสามารถเลือกวิธีการจัดการสถานการณ์นั้น คุณสามารถเปลี่ยนลักษณะการทำงานของภาพ โดยไปลงในการ**บานหน้าต่างการจัดรูปแบบ**และสลับไปมาระหว่าง**ชนิดวิเคราะห์แบบกลุ่ม**และ**ชนิดวิเคราะห์แบบต่อเนื่อง**ได้

![เปลี่ยนจากประเภทแบบต่อเนื่อง](media/power-bi-visualization-influencers/power-bi-ki-formatting.png)

A **ชนิดวิเคราะห์แบบกลุ่ม**ลักษณะการทำงานตามที่อธิบายไว้ข้างต้น เช่น ถ้าคุณกำลังดูสำรวจคะแนนตั้งแต่ 1 ถึง 10 คุณสามารถสอบถาม 'ที่มีผลต่อสำรวจคะแนนเป็น 1 ได้อย่างไร'

A **ชนิดการวิเคราะห์แบบต่อเนื่อง**เปลี่ยนคำถามเป็นแบบต่อเนื่อง ในตัวอย่างด้านบน คำถามใหม่ของเราจะเป็น 'สิ่งมีผลต่อคะแนนสำรวจเพื่อเพิ่ม/ลดลง ได้อย่างไร'

ความแตกต่างนี้จะมีประโยชน์มากเมื่อคุณมีหลายค่าที่ไม่ซ้ำกันในเขตข้อมูลคุณกำลังวิเคราะห์อยู่นั้น ในตัวอย่างด้านล่าง เราดูที่บ้านราคา ไม่มีความหมายมากถาม 'ที่มีผลต่อราคาบ้านต้อง 156,214 ' ที่มีมาก เฉพาะและเราจะไม่มีข้อมูลเพียงพอที่จะสรุปแบบ

เราอาจต้องการถาม 'ที่มีผลต่อราคาบ้านเพื่อเพิ่ม' หรือไม่ ซึ่งช่วยให้เราจะถือว่าราคาบ้านช่วงแทนที่ค่าเฉพาะ

![คำถามตัวเลข](media/power-bi-visualization-influencers/power-bi-ki-numeric-question.png)

## <a name="interpret-the-results-key-influencers"></a>แปลงผลลัพธ์: ผู้มีอิทธิพลที่สำคัญ 

ในสถานการณ์นี้ เราดู 'ที่มีผลต่อราคาบ้านเพื่อเพิ่ม' เรากำลังดูจำนวนอธิบายปัจจัยที่อาจส่งผลกระทบต่อราคาในบ้านเช่น**สร้างปี** (ปีบ้านถูกสร้างขึ้น), **KitchenQual** (ห้องครัวคุณภาพ) และ**YearRemodAdd** (ปีบ้านถูกปรับรูปแบบ) 

ในตัวอย่างด้านล่าง เราดูที่ influencer ของเราด้านบนซึ่งเป็นห้องครัวคุณภาพดีเยี่ยม ผลลัพธ์จะคล้ายกับเราเห็นเมื่อเราได้วิเคราะห์เมตริก มีความแตกต่างที่สำคัญสองสามประเภท:

- แผนภูมิคอลัมน์ทางด้านขวากำลังดูค่าเฉลี่ยแทนเปอร์เซ็นต์ ดังนั้นแสดงเราสิ่งราคาบ้านโดยเฉลี่ยของบ้านกับห้องครัวดีเยี่ยมเป็น (สีเขียวแถบ) เมื่อเทียบกับราคาบ้านโดยเฉลี่ยของบ้านโดยไม่มีห้องครัวดีเยี่ยม (เส้นประสี)
- ตัวเลขในแผนภูมิฟองยังคงความแตกต่างระหว่างเส้นสีแดงและสีเขียวแถบ แต่จะแสดงเป็นตัวเลข ($158 49K) แทนที่จะเป็นความน่าเป็น (1.93 x) บนเฉลี่ย บ้านหลังหนึ่งกับ kitchens ดีเยี่ยมจะเกือบ $160K แพงกว่าบ้านหลังหนึ่งโดยไม่ต้อง kitchens ดีเยี่ยม

![เป้าหมายตัวเลข influencers แบบกลุ่ม](media/power-bi-visualization-influencers/power-bi-ki-numeric-categorical.png)

ในตัวอย่างด้านล่างเรากำลังดูผลกระทบ ตัวอย่างต่อเนื่อง (ปีบ้านถูกปรับรูปแบบ) มีราคาบ้าน ความแตกต่างเมื่อเทียบกับวิธีที่เราวิเคราะห์ influencers อย่างต่อเนื่องสำหรับเมตริกประเภทมีดังนี้:

-   ลงจุดแผนภูมิกระจายในบานหน้าต่างด้านขวาลงจุดราคาบ้านเฉลี่ยสำหรับแต่ละค่าที่แตกต่างกันของปีที่ปรับรูปแบบ 
-   แสดงค่าในแผนภูมิฟอง โดยปริมาณบ้านเฉลี่ยราคาเพิ่ม (ในกรณีนี้ $2 87k) เมื่อปีที่บ้านถูกปรับรูปแบบเพิ่ม ด้วยค่าเบี่ยงเบนมาตรฐานของ (ในกรณีนี้ 20 ปี)

![เป้าหมายตัวเลข influencers อย่างต่อเนื่อง](media/power-bi-visualization-influencers/power-bi-ki-numeric-continuous.png)

สุดท้าย ในกรณีของหน่วยวัดที่เรากำลังดูปีเฉลี่ย บ้านถูกสร้างขึ้น การวิเคราะห์ที่นี่จะเป็นดังนี้:

-   Scatterplot ในบานหน้าต่างด้านขวาลงจุดราคาบ้านเฉลี่ยสำหรับแต่ละค่าที่แตกต่างกันในตาราง
-   แสดงค่าในแผนภูมิฟอง โดยปริมาณบ้านเฉลี่ยราคาเพิ่ม (ในกรณีนี้ $1 35K) เมื่อเพิ่มปีเฉลี่ยของค่าเบี่ยงเบนมาตรฐาน (ในกรณีนี้ 30 ปี)

![Influencers ไม่พบข้อผิดพลาด](media/power-bi-visualization-influencers/power-bi-error1.png)


ข้อผิดพลาดนี้เกิดขึ้นเมื่อคุณรวมเขตข้อมูลใน**อธิบายโดย**แต่พบ influencers ไม่ 
- คุณรวมการวัดที่คุณกำลังวิเคราะห์อยู่นั้นอยู่ในทั้งสอง**วิเคราะห์**และ**อธิบายโดย** เอาออกจาก**อธิบายโดย** 
- เขตข้อมูลการอธิบายของคุณมีหมวดหมู่มากเกินไปโดยมีข้อสังเกตเพียงเล็กน้อย สถานการณ์นี้ทำให้ยากสำหรับการแสดงภาพเพื่อดูปัจจัยที่ influencers ยากที่จะยึดเพียงไม่กี่สังเกตทั่วไป ถ้าคุณกำลังวิเคราะห์เขตข้อมูลตัวเลข คุณอาจต้องการสลับจาก**วิเคราะห์แบบกลุ่ม**เพื่อ**วิเคราะห์แบบต่อเนื่อง**ในการ**บานหน้าต่างการจัดรูปแบบ**ภายใต้การ **วิเคราะห์**การ์ด
- ปัจจัยของคุณอธิบายมีเพียงพอให้สังเกตทั่วไป แต่แสดงภาพไม่พบความสัมพันธ์ใด ๆ สื่อความหมายกับรายงาน
 
**ฉันเห็นข้อผิดพลาดการวัดฉันวิเคราะห์ไม่มีข้อมูลเพียงพอเพื่อเรียกใช้การวิเคราะห์ เหตุใดจึงเป็นเช่นนั้น** 

![ข้อผิดพลาดข้อมูลไม่เพียงพอ](media/power-bi-visualization-influencers/power-bi-not-enough-data.png)

แสดงภาพทำงาน ด้วยการดูรูปแบบในข้อมูลสำหรับกลุ่มหนึ่งเมื่อเทียบกับกลุ่มอื่น ๆ ตัวอย่าง จะค้นหาลูกค้าที่ให้การจัดอันดับต่ำเมื่อเปรียบเทียบกับลูกค้าที่กำหนดให้กับการจัดอันดับสูง ถ้าข้อมูลในแบบจำลองของคุณมีเพียงไม่กี่สังเกต รูปแบบเป็นเรื่องยากที่จะค้นหา ถ้าการแสดงภาพไม่มีข้อมูลเพียงพอเพื่อค้นหาสื่อความหมาย influencers บ่งชี้ว่า ข้อมูลเพิ่มเติมที่จำเป็นต้องเรียกใช้การวิเคราะห์ 

เราขอแนะนำว่า คุณมีอย่างน้อย 100 สังเกตสำหรับสถานะเลือกไว้ ในกรณีนี้ สถานะคือ ลูกค้าที่ churn นอกจากนี้คุณยังต้องสังเกตน้อย 10 สำหรับสถานะคุณใช้สำหรับการเปรียบเทียบ ในกรณีนี้ สถานะการเปรียบเทียบคือ ลูกค้าที่ไม่ churn

ถ้าคุณกำลังวิเคราะห์เขตข้อมูลตัวเลข คุณอาจต้องการสลับจาก**วิเคราะห์แบบกลุ่ม**เพื่อ**วิเคราะห์แบบต่อเนื่อง**ในการ**บานหน้าต่างการจัดรูปแบบ**ภายใต้การ **วิเคราะห์**การ์ด

**ฉันเห็นข้อผิดพลาดที่เขตข้อมูลใน*อธิบายโดย*ไม่เกี่ยวข้องกับตารางที่ประกอบด้วยการวัดฉันวิเคราะห์แบบไม่ซ้ำกัน เหตุใดจึงเป็นเช่นนั้น**
 
การวิเคราะห์การทำงานในระดับตารางของเขตข้อมูลที่กำลังวิเคราะห์ ตัวอย่าง ถ้าคุณวิเคราะห์คำติชมของลูกค้าสำหรับบริการของคุณ คุณอาจมีตารางที่บอกคุณว่าลูกค้าให้กับการจัดอันดับสูงหรือการจัดอันดับต่ำ ในกรณีนี้ การวิเคราะห์ของคุณใช้งานในระดับตารางลูกค้า 

ถ้าคุณมีตารางที่เกี่ยวข้องที่กำหนดไว้ในระดับมากกว่าตารางที่ประกอบด้วยการวัดของคุณโดยละเอียด คุณเห็นข้อผิดพลาดนี้ นี่คือตัวอย่าง: 
 
- คุณวิเคราะห์ลูกค้าเพื่อให้จัดอันดับต่ำของบริการไดรฟ์ใด
- คุณต้องการดูถ้าอุปกรณ์ที่ลูกค้ากำลังใช้บริการของคุณมีผลต่อบทวิจารณ์ที่พวกเขาให้
- ลูกค้าสามารถใช้บริการในหลายวิธี
- ในตัวอย่างต่อไปนี้ ลูกค้า 10000000 ใช้เบราว์เซอร์และแท็บเล็ตทั้งเพื่อโต้ตอบกับบริการ

![ตารางเกี่ยวข้องที่กำหนดไว้ในระดับแยกย่อยยิ่งกว่าตารางที่ประกอบด้วยการวัดของคุณ](media/power-bi-visualization-influencers/power-bi-error2.png)

ถ้าคุณพยายามจะใช้คอลัมน์อุปกรณ์เป็นปัจจัยการอธิบาย คุณเห็นข้อผิดพลาดต่อไปนี้: 

![ข้อผิดพลาดคอลัมน์ไม่ถูกต้อง](media/power-bi-visualization-influencers/power-bi-error3.png)

ข้อผิดพลาดนี้ปรากฏขึ้นเนื่องจากอุปกรณ์ไม่ได้กำหนดระดับลูกค้า ลูกค้าหนึ่งสามารถใช้บริการบนอุปกรณ์หลายเครื่อง สำหรับการแสดงภาพเพื่อค้นหารูปแบบ อุปกรณ์ต้องเป็นแอตทริบิวต์ของลูกค้า มีวิธีแก้ไขปัญหาหลากหลายที่ขึ้นอยู่กับความเข้าใจเกี่ยวกับธุรกิจของคุณ: 
 
- คุณสามารถเปลี่ยนข้อสรุปของอุปกรณ์การนับ ตัวอย่าง ใช้จำนวนถ้าจำนวนอุปกรณ์ที่อาจส่งผลต่อคะแนนที่ทำให้ลูกค้า 
- คุณสามารถทำ pivot คอลัมน์อุปกรณ์เพื่อดูว่า การใช้บริการบนอุปกรณ์บางเครื่องมีผลต่อการจัดอันดับของลูกค้า
 
ในตัวอย่างนี้ ข้อมูลถูก pivoted เพื่อสร้างคอลัมน์ใหม่สำหรับเบราว์เซอร์ อุปกรณ์เคลื่อนที่ และแท็บเล็ต คุณสามารถใช้อุปกรณ์เหล่านี้เฉพาะใน**อธิบายโดย**ได้ อุปกรณ์ทั้งหมดเปิดออกเป็น influencers และเบราว์เซอร์ที่มีผลมากที่สุดบนคะแนนลูกค้า

เพิ่มเติมได้อย่างแม่นยำ ลูกค้าที่ไม่ได้ใช้เบราว์เซอร์เพื่อใช้บริการครั้ง 3.79 ดูเหมือนว่าจะให้คะแนนต่ำกว่าลูกค้าที่ทำ ด้านล่างลงในรายการ สำหรับมือถือผกผันเป็นจริง ลูกค้าที่ใช้แอปสำหรับอุปกรณ์เคลื่อนให้คะแนนต่ำกว่าลูกค้าที่ไม่ได้

![หน่วยวัดไม่รวมข้อผิดพลาด](media/power-bi-visualization-influencers/power-bi-measures-not-included.png)


การวิเคราะห์การทำงานในระดับตารางของเขตข้อมูลที่กำลังวิเคราะห์ ถ้าคุณวิเคราะห์ churn ลูกค้า คุณอาจมีตารางที่บอกคุณว่า ลูกค้า churned หรือไม่ ในกรณีนี้ การวิเคราะห์ของคุณทำงานในระดับตารางลูกค้า
 
หน่วยวัดและรวมได้ตามค่าเริ่มต้นในการวิเคราะห์ระดับตาราง ถ้าไม่มีหน่วยวัดสำหรับค่าใช้จ่ายรายเดือนโดยเฉลี่ย นั้นจะสามารถวิเคราะห์ระดับตารางลูกค้า 

ถ้าตารางลูกค้าไม่มีตัวระบุที่ไม่ซ้ำกัน คุณไม่สามารถประเมินหน่วยวัด และนั้นจะถูกละเว้น โดยการวิเคราะห์ เมื่อต้องการหลีกเลี่ยงเหตุการณ์เช่นนี้ ให้แน่ใจว่าตารางที่ มีการวัดของคุณมีตัวระบุที่ไม่ซ้ำกัน ในกรณีนี้ ตารางลูกค้า และตัวระบุที่ไม่ซ้ำกันเป็น id ลูกค้า นอกจากนี้ยังเป็นเรื่องง่ายเมื่อต้องเพิ่มคอลัมน์ดัชนี โดยใช้ Power Query
 
**ฉันเห็นคำเตือนว่า เมตริกฉันวิเคราะห์มีค่าไม่ซ้ำกันมากกว่า 10 และว่า จำนวนนี้อาจส่งผลต่อคุณภาพของการวิเคราะห์ของฉัน เหตุใดจึงเป็นเช่นนั้น** 

แสดงภาพ AI สามารถวิเคราะห์เขตข้อมูลประเภทและเขตข้อมูลตัวเลข ในกรณีของเขตข้อมูลประเภท ตัวอย่างอาจเป็น Churn ใช่หรือไม่ และความพึงพอใจของลูกค้าเป็นสูง ปานกลาง หรือต่ำได้ เพิ่มจำนวนประเภทเพื่อวิเคราะห์หมายความว่า มีสังเกตน้อยลงสำหรับแต่ละประเภท สถานการณ์นี้ทำให้ยากต่อการแสดงภาพเพื่อค้นหารูปแบบในข้อมูล 

เมื่อวิเคราะห์เขตข้อมูลตัวเลข คุณสามารถเลือกระหว่างถือเขตข้อมูลตัวเลขเช่นข้อความในกรณี คุณจะเรียกใช้การวิเคราะห์เดียวกันเหมือนที่คุณทำกับข้อมูลประเภท (**วิเคราะห์แบบกลุ่ม**) ถ้าคุณมีจำนวนมากที่แตกต่างกัน ค่าเราขอแนะนำให้คุณสลับการวิเคราะห์การ**วิเคราะห์แบบต่อเนื่อง**เป็นซึ่งหมายความว่า เราสามารถอ้างถึงรูปแบบจากเมื่อตัวเลขเพิ่ม หรือลดแทนที่ปฏิบัติตามค่าที่ไม่ คุณสามารถสลับจาก**วิเคราะห์แบบกลุ่ม**เพื่อ**วิเคราะห์แบบต่อเนื่อง**ในการ**บานหน้าต่างการจัดรูปแบบ**ภายใต้**Analysis**การ์ดได้

เมื่อต้องการค้นหาอย่างรัดกุม influencers เราขอแนะนำให้ คุณจัดกลุ่มค่าคล้ายกันเป็นหน่วยเดียว ตัวอย่าง ถ้าคุณมีเมตริกสำหรับราคา คุณมักจะได้รับผลลัพธ์ที่ดีขึ้น โดยจัดกลุ่มราคาที่คล้ายกันเป็นสูง ปานกลาง ต่ำประเภท และเปรียบเทียบกับโดยใช้จุดราคาแต่ละรายการ 

![คำเตือนปัจจัยไม่ซ้ำกันมากกว่า 10](media/power-bi-visualization-influencers/power-bi-error4.png)


**มีปัจจัยในข้อมูลของฉันที่มีลักษณะเหมือนพวกเขาควร influencers คีย์ แต่พวกเขาไม่ สถานการณ์เช่นนั้นเกิดขึ้นได้อย่างไร**

ในตัวอย่างต่อไปนี้ ลูกค้าที่เป็นผู้บริโภคไดรฟ์จัดอันดับต่ำ ด้วย 14.93% ของอันดับที่ต่ำ บทบาทผู้ดูแลยังมีสัดส่วนของการจัดอันดับต่ำ, % 13.42 สูง แต่จะไม่ถือว่าเป็น influencer 

เหตุผลสำหรับการกำหนดนี้คือ ที่แสดงภาพยังพิจารณาจำนวนของจุดข้อมูลเมื่อพบ influencers ตัวอย่างต่อไปนี้มีลูกค้ามากกว่า 29,000 และผู้ดูแลระบบน้อย 10 ครั้ง เกี่ยวกับ 2,900 390 เฉพาะของพวกเขาให้กับการจัดอันดับต่ำ วิชวลไม่มีข้อมูลเพียงพอเพื่อกำหนดว่า จะพบรูปแบบการจัดอันดับผู้ดูแลระบบ หรือถ้ามีโอกาสเพียงค้นหา

**วิธีใดคุณจึงทำคำนวณ influencers คีย์สำหรับการวิเคราะห์ตามประเภท**

ในเบื้องหลัง ใช้การแสดงภาพ AI [ML.NET](https://dotnet.microsoft.com/apps/machinelearning-ai/ml-dotnet)เพื่อเรียกใช้การถดถอยลอจิสติกส์เพื่อคำนวณ influencers คีย์ การถดถอยโลจิสติกเป็นรูปแบบทางสถิติที่เปรียบเทียบกลุ่มที่แตกต่างกัน 

ถ้าคุณต้องการดูการจัดอันดับต่ำไดรฟ์ใด การถดถอยลอจิสติกส์ดูที่วิธีที่ลูกค้าที่ให้คะแนนต่ำแตกต่างจากลูกค้าที่ให้คะแนนสูงสุด ถ้าคุณมีหลายประเภท เช่นคะแนนสูง กลาง และสาย คุณดูวิธีการที่ลูกค้าที่กำหนดให้กับการจัดอันดับต่ำแตกต่างจากลูกค้าที่ไม่ได้ให้การจัดอันดับต่ำ ในกรณีนี้ วิธีทำลูกค้าที่ให้คะแนนต่ำแตกต่างจากลูกค้าที่กำหนดให้กับการจัดอันดับสูงหรือการจัดอันดับกลางอย่างไร 
 
การถดถอยลอจิสติกส์ค้นหารูปแบบในข้อมูล และค้นหาวิธีที่ลูกค้าที่กำหนดให้กับการจัดอันดับต่ำอาจแตกต่างจากลูกค้าที่กำหนดให้กับการจัดอันดับสูง ระบบอาจพบ ตัวอย่าง ว่า ลูกค้าที่ มีใบอนุญาตเพิ่มเติมให้เปอร์เซ็นต์ที่สูงกว่าของการจัดอันดับต่ำกว่าลูกค้าที่มีใบอนุญาตเพียงเล็กน้อย หรือไม่
 
การถดถอยลอจิสติกส์ยังพิจารณาจำนวนจุดข้อมูลอยู่ ตัวอย่าง ถ้าลูกค้าที่มีบทบาทผู้ดูแลระบบให้คะแนนสัดส่วนลบมาก แต่มีเพียงไม่กี่ผู้ดูแลระบบ ปัจจัยนี้ไม่ได้พิจารณา influential การกำหนดนี้ไว้เนื่องจากไม่มีจุดข้อมูลเพียงพอเพื่อสรุปแบบที่พร้อมใช้งาน การทดสอบทางสถิติ เรียกว่าการทดสอบ Wald จะใช้เพื่อกำหนดว่า ตัวคูณจะถือว่าเป็น influencer ภาพวิชวลใช้ค่า p เป็น 0.05 เพื่อกำหนดเกณฑ์ 

**วิธีใดคุณจึงทำคำนวณ influencers คีย์สำหรับการวิเคราะห์ตัวเลข**

ในเบื้องหลัง ใช้การแสดงภาพ AI [ML.NET](https://dotnet.microsoft.com/apps/machinelearning-ai/ml-dotnet)เพื่อเรียกใช้การถดถอยเชิงเส้นเพื่อคำนวณ influencers คีย์ แบบจำลองทางสถิติที่ดูที่การเปลี่ยนแปลงผลลัพธ์ของเขตข้อมูลคุณกำลังวิเคราะห์อยู่นั้นขึ้นอยู่กับปัจจัยของคุณอธิบาย ได้การถดถอยเชิงเส้น

ตัวอย่าง ถ้าเราจะวิเคราะห์ราคาบ้าน การถดถอยเชิงเส้นจะดูที่การผลกระทบมีห้องครัวดีเยี่ยมจะมีราคาบ้าน บ้านหลังหนึ่งกับ kitchens ดีเยี่ยมโดยทั่วไปแล้วจะบ้านสูง หรือต่ำกว่าราคาเมื่อเทียบกับบ้านหลังหนึ่งโดยไม่ต้อง kitchens ดีมากหรือไม่

การถดถอยเชิงเส้นยังพิจารณาจำนวนของจุดข้อมูล ตัวอย่าง ถ้าบ้านหลังหนึ่งกับศาลเทนนิสมีราคาสูง แต่เรามีบ้านหลังหนึ่งไม่มากซึ่งมีสนามเทนนิส ปัจจัยนี้จะไม่ถือเป็น influential การกำหนดนี้ไว้เนื่องจากไม่มีจุดข้อมูลเพียงพอเพื่อสรุปแบบที่พร้อมใช้งาน การทดสอบทางสถิติ เรียกว่าการทดสอบ Wald จะใช้เพื่อกำหนดว่า ตัวคูณจะถือว่าเป็น influencer ภาพวิชวลใช้ค่า p เป็น 0.05 เพื่อกำหนดเกณฑ์ 

**คุณคำนวณกลุ่มอย่างไร**

ในเบื้องหลัง ใช้การแสดงภาพ AI [ML.NET](https://dotnet.microsoft.com/apps/machinelearning-ai/ml-dotnet)เพื่อเรียกใช้ทรีการตัดสินใจเพื่อค้นหาย่อยที่น่าสนใจ วัตถุประสงค์ของทรีการตัดสินใจให้ สิ้นสุด ด้วยกลุ่มย่อยของจุดข้อมูลที่ค่อนข้างสูงในการวัดที่คุณสนใจ ได้ ซึ่งอาจเป็นลูกค้าที่ มีการจัดอันดับต่ำหรือบ้านหลังหนึ่ง ด้วยราคาสูง

ทรีการตัดสินใจที่จะคูณแต่ละอธิบาย และพยายามเหตุผลปัจจัยใดให้ดีที่สุด*แยก* ตัวอย่าง ถ้าคุณกรองข้อมูลที่จะรวมเฉพาะลูกค้าองค์กรขนาดใหญ่ จะที่แยกออกลูกค้าที่กำหนดให้กับการจัดอันดับสูงเทียบกับการจัดอันดับต่ำหรือไม่ หรืออาจเป็นได้ดียิ่งขึ้นเพื่อกรองข้อมูลที่จะรวมเฉพาะลูกค้าที่แสดงข้อคิดเห็นเกี่ยวกับความปลอดภัยหรือไม่ 

หลังจากทรีการตัดสินใจไม่การแยก ใช้กลุ่มย่อยของข้อมูล และกำหนดแยกที่ดีที่สุดถัดไปสำหรับข้อมูลนั้น ในกรณีนี้ กลุ่มย่อยคือ ลูกค้าที่แสดงข้อคิดเห็นบนความปลอดภัย หลังจากแต่ละแยก นั้นยังพิจารณาว่า มีจุดข้อมูลเพียงพอสำหรับกลุ่มนี้เป็น ตัวแทนพอเพื่อสรุปจากรูปแบบหรือไม่ว่าจะมีสิ่งผิดปกติในข้อมูลและไม่เซ็กเมนต์จริง การทดสอบทางสถิติอื่นถูกนำไปใช้เพื่อตรวจสอบค่านัยสำคัญทางสถิติของเงื่อนไขแยกด้วย p-ค่า 0.05 

หลังจากที่ทำงานเสร็จทรีการตัดสินใจ ใช้ทั้งหมดแยก เช่นข้อคิดเห็นความปลอดภัยและองค์กรขนาดใหญ่ และสร้างตัวกรอง Power BI มีการรวมแพคเกจชุดตัวกรองเป็นกลุ่มในภาพวิชวล 
 
**ทำไมต้องทำบางปัจจัยเป็น influencers หรือหยุดรับ influencers เมื่อฉันย้ายเขตข้อมูลเพิ่มเติมลงในการ*อธิบายโดย*เขตข้อมูลหรือไม่**

การแสดงภาพข้อมูลประเมินปัจจัยอธิบายทั้งหมดเข้าด้วยกัน ปัจจัยอาจมี influencer ด้วยตัวเอง แต่เมื่อถือเป็นกับปัจจัยอื่นๆ อาจไม่ สมมติว่าคุณต้องการวิเคราะห์ราคาในบ้านเป็นสูง กับห้องนอนและขนาดบ้านเป็นปัจจัยอธิบายไดรฟ์ใด:

- ด้วยตัวเอง ห้องนอนเพิ่มเติมอาจมีโปรแกรมควบคุมสำหรับบ้านราคาเป็นสูง
- ขนาดบ้านรวมถึงในการวิเคราะห์หมายความว่า คุณตอนนี้มาดูเกิดอะไรขึ้นกับห้องนอนขณะขนาดบ้านยังคงคง
- ถ้าขนาดบ้านได้รับการแก้ไขที่ตารางฟุต 1500 ไม่น่าที่เพิ่มขึ้นอย่างต่อเนื่องในหมายเลขของห้องนอนทำเพิ่มราคาที่บ้าน 
- ห้องนอนอาจไม่จำเป็นของปัจจัยยุ่งยากก่อนถูกพิจารณาว่าเป็นขนาดบ้าน
